# DSL-169: run_epoch_boundary Wires reward_payout for FlagDelta Rewards

> **Authoritative:** [DSL-169](../NORMATIVE.md#DSL-169) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §10, §8.4

---

## Summary

`run_epoch_boundary` currently computes `flag_deltas: Vec<FlagDelta>` but discards the `reward_payout: &mut dyn RewardPayout` parameter (`let _ = &reward_payout;` at src/orchestration.rs:191). Close the wiring: for every `FlagDelta { validator_index, reward, penalty }` with `reward > 0`, call `RewardPayout::pay(validator_set.get(idx)?.puzzle_hash(), reward)` in-line with the existing step 1. Penalties are already debited via downstream slash_absolute on the validator; rewards need an explicit payout routing.

Also: inactivity-leak penalty payouts — for every `(idx, penalty)` in `inactivity_penalties`, call `validator_set.get_mut(idx)?.slash_absolute(penalty, current_epoch_ending)` so the penalty actually debits the validator's balance (currently the vec is computed and returned but never applied).

---

## Specification

Inside `run_epoch_boundary`, after step 1 (flag deltas computed) and step 3 (inactivity penalties computed), add:

```rust
// ── Step 1b: route flag-delta rewards through RewardPayout ───
for fd in &flag_deltas {
    if fd.reward == 0 {
        continue;
    }
    if let Some(entry) = validator_set.get(fd.validator_index) {
        reward_payout.pay(entry.puzzle_hash(), fd.reward);
    }
    // Index out of view → skip (defensive).
}

// ── Step 3b: apply inactivity-leak penalties to validator stake ─
for &(idx, penalty_mojos) in &inactivity_penalties {
    if let Some(entry) = validator_set.get_mut(idx) {
        entry.slash_absolute(penalty_mojos, current_epoch_ending);
    }
}
```

Step 1b runs immediately after step 1; step 3b runs immediately after step 3. Does NOT change step numbering or the existing 8-step fixed order pinned by DSL-127.

---

## Acceptance Criteria

- [ ] For each FlagDelta with `reward > 0`, `RewardPayout::pay(puzzle_hash, reward)` is called exactly once.
- [ ] FlagDelta with `reward == 0` does NOT trigger a `pay` call.
- [ ] For each `(idx, penalty)` in `inactivity_penalties`, `slash_absolute(penalty, epoch)` is called exactly once.
- [ ] FlagDelta.penalty field is untouched by this wiring (documented as computed-for-observability; slash routing lives on the validator trait, not here).
- [ ] Validator missing from `validator_set.get(idx)` → silent skip for rewards AND penalties.
- [ ] EpochBoundaryReport fields unchanged — this is behaviour wiring, not a new field.
