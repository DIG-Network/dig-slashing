# DSL-170: SlashingSystem Carries network_id

> **Authoritative:** [DSL-170](../NORMATIVE.md#DSL-170) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §11

---

## Summary

`GenesisParameters::network_id` is currently reserved and unused — `SlashingSystem::genesis` accepts it in the struct but never stores it on `SlashingSystem`. Close the gap: store `network_id: Bytes32` on `SlashingSystem` and expose `SlashingSystem::network_id(&self) -> &Bytes32`. Downstream admission entrypoints (DSL-168 `process_block_admissions`, domain-separated signature verify) can read it from the system aggregate rather than requiring callers to thread it through every argument list.

---

## Specification

```rust
#[derive(Debug)]
pub struct SlashingSystem {
    pub manager: SlashingManager,
    pub participation: ParticipationTracker,
    pub inactivity: InactivityScoreTracker,
    network_id: Bytes32,
}

impl SlashingSystem {
    #[must_use]
    pub fn genesis(params: &GenesisParameters) -> Self {
        Self {
            manager: SlashingManager::new(params.genesis_epoch),
            participation: ParticipationTracker::new(
                params.initial_validator_count,
                params.genesis_epoch,
            ),
            inactivity: InactivityScoreTracker::new(params.initial_validator_count),
            network_id: params.network_id,
        }
    }

    #[must_use]
    pub fn network_id(&self) -> &Bytes32 {
        &self.network_id
    }
}
```

---

## Acceptance Criteria

- [ ] `SlashingSystem::genesis(params)` stores `params.network_id` verbatim.
- [ ] `SlashingSystem::network_id()` returns the exact bytes supplied at genesis.
- [ ] Field is private; only accessor reads it (no direct `sys.network_id` access).
- [ ] Does NOT change `GenesisParameters` shape.
- [ ] Does NOT change `SlashingManager::submit_evidence` signature — it still takes `network_id: &Bytes32` as an explicit arg (embedder passes `sys.network_id()`).
- [ ] SlashingSystem remains `Debug`-derivable (inner bytes formatted via Bytes32's existing Display).

---

## Implementation Notes

- **Primary file:** `src/system.rs`
- No new dependency. No new trait. No change to manager/tracker construction.
- Future work (not in this DSL): `SlashingSystem::submit_evidence(&mut self, evidence, ...)` convenience wrapper that reads `self.network_id()` — tracked separately if needed.
