# DSL-165: EpochBoundaryReport + ReorgReport + FlagDelta Serde Roundtrip

> **Authoritative:** [DSL-165](../NORMATIVE.md#DSL-165) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §10, §13, §18

---

## Summary

Three orchestration result types MUST round-trip byte-exactly via bincode + serde_json:

- `EpochBoundaryReport` (flag_deltas + inactivity_penalties + finalisations + in_finality_stall)
- `ReorgReport` (rewound_pending_slashes + participation_epochs_dropped + inactivity_epochs_dropped + protection_rewound)
- `FlagDelta` (validator_index + reward + penalty)

Required for telemetry, snapshot/restore, and RPC responses.

---

## Specification

All three types `derive(Serialize, Deserialize, PartialEq, Eq)`.

---

## Acceptance Criteria

- [ ] `EpochBoundaryReport` round-trips (populated vecs + stall flag).
- [ ] `ReorgReport` round-trips (populated hash list + counts + flag).
- [ ] `FlagDelta` individual round-trips.
- [ ] Empty-vec cases preserved.
- [ ] `in_finality_stall` true/false both preserved.
