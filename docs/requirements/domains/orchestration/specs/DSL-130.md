# DSL-130: rewind_all_on_reorg + ReorgTooDeep

> **Authoritative:** [DSL-130](../NORMATIVE.md#DSL-130) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §13, §22.15

---

## Summary

`rewind_all_on_reorg(...)` is the global reorg orchestrator. Rewinds (in order):
1. `SlashingManager::rewind_on_reorg` (DSL-129)
2. `ParticipationTracker::rewind_on_reorg`
3. `InactivityScoreTracker::rewind_on_reorg`
4. `SlashingProtection::reconcile_with_chain_tip` (DSL-099)

Returns a `ReorgReport`. Rewinds deeper than `CORRELATION_WINDOW_EPOCHS` (36) MUST return `SlashingError::ReorgTooDeep` — the trackers only retain snapshots back that far.

---

## Specification

```rust
pub fn rewind_all_on_reorg(
    manager: &mut SlashingManager,
    participation: &mut ParticipationTracker,
    inactivity: &mut InactivityScoreTracker,
    protection: &mut SlashingProtection,
    validator_set: &mut dyn ValidatorView,
    collateral: Option<&mut dyn CollateralSlasher>,
    bond_escrow: &mut dyn BondEscrow,
    new_tip_epoch: u64,
    new_tip_slot: u64,
    validator_count: usize,
) -> Result<ReorgReport, SlashingError>;

pub struct ReorgReport {
    pub rewound_pending_slashes: Vec<Bytes32>,
    pub participation_epochs_dropped: u64,
    pub inactivity_epochs_dropped: u64,
    pub protection_rewound: bool,
}
```

---

## Acceptance Criteria

- [ ] All four subsystems rewound (manager, participation, inactivity, protection).
- [ ] `ReorgReport` populated with rewind counts.
- [ ] Depth > 36 → `SlashingError::ReorgTooDeep`.
- [ ] Depth == 36 → accepted (boundary).
- [ ] Ordering: manager first (may consult validator state untouched by other rewinds).
