# DSL-129: SlashingManager::rewind_on_reorg

> **Authoritative:** [DSL-129](../NORMATIVE.md#DSL-129) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §13, §22.15

---

## Summary

On fork-choice reorg to `new_tip_epoch`, `SlashingManager::rewind_on_reorg` MUST, for every `PendingSlash` with `submitted_at_epoch > new_tip_epoch`:
- Credit the base slash back via `ValidatorEntry::credit_stake`.
- Restore collateral via `CollateralSlasher::credit`.
- Call `ValidatorEntry::restore_status`.
- Release the reporter bond in full via `BondEscrow::release` (NO forfeit, NO reporter penalty — reorg is not the reporter's fault).
- Remove the entry from `processed` and `slashed_in_window`.
- Return the list of rewound `evidence_hash` values.

```rust
pub fn rewind_on_reorg(

    validator_set: &mut dyn ValidatorView,
    collateral: Option<&mut dyn CollateralSlasher>,
    bond_escrow: &mut dyn BondEscrow,

```

## Acceptance Criteria

- [ ] Every PendingSlash with `submitted_at_epoch > new_tip_epoch` is rewound.
- [ ] `credit_stake` called per slashed validator.
- [ ] `restore_status` called per slashed validator.
- [ ] `release` called on reporter bond (not forfeit).
- [ ] Entries removed from `processed` and `slashed_in_window`.
- [ ] Rewound hashes returned.
- [ ] NO reporter penalty applied (distinct from sustained-appeal path in DSL-069).
