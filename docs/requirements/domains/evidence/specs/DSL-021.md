## Summary

`verify_evidence_for_inclusion` is the mempool-admission wrapper around `verify_evidence`. It runs the full per-offense verifier chain but performs NO state mutation. For identical inputs, the two functions MUST produce identical verdicts.

Used by `dig-mempool` during REMARK admission (before a block containing this evidence is accepted). Decoupled from `SlashingManager` to keep the mempool's `&dyn ValidatorView` access pattern read-only.

---

## Specification

```rust
pub fn verify_evidence_for_inclusion(
    evidence: &SlashingEvidence,
    validator_view: &dyn ValidatorView,
    network_id: &Bytes32,
    current_epoch: u64,
) -> Result<VerifiedEvidence, SlashingError>;
```

Implementation is identical to `verify_evidence` except:
- No `&mut` access required.
- No `processed` map check (deduplication happens at REMARK mempool layer).

---

## Acceptance Criteria

- [ ] Identical inputs → identical `Result<VerifiedEvidence, SlashingError>`.
- [ ] No state mutation in `validator_view`.
- [ ] Takes `&dyn ValidatorView` (not `&mut`).
- [ ] Runs every check in DSL-011..020 except the processed-map dedup.
