## Summary

`SlashingEvidence::hash()` is the content-addressed identity of an evidence envelope. `SlashingManager` uses it as the `processed` map key and as the bond-escrow tag. Two honest nodes computing `hash()` for the same `SlashingEvidence` MUST agree bit-for-bit. Any mutation of the envelope MUST shift the hash.

The hash is SHA-256 over the `DOMAIN_SLASHING_EVIDENCE` domain tag concatenated with the `bincode`-serialized envelope. `chia_sha2::Sha256` is the hasher.

---

## Specification

```rust
impl SlashingEvidence {
    pub fn hash(&self) -> Bytes32 {
        let mut h = Sha256::new();
        h.update(DOMAIN_SLASHING_EVIDENCE);
        h.update(&bincode::serialize(self).unwrap());
        Bytes32::new(h.finalize().into())
    }
}
```

---

## Acceptance Criteria

- [ ] Repeated calls on the same `SlashingEvidence` return bit-identical `Bytes32`.
- [ ] Mutating `offense_type`, `reporter_validator_index`, `reporter_puzzle_hash`, `epoch`, or any byte of `payload` changes the hash.
- [ ] Domain-prefixed by `DOMAIN_SLASHING_EVIDENCE`.
- [ ] Uses `chia_sha2::Sha256` (not `sha2::Sha256`).
