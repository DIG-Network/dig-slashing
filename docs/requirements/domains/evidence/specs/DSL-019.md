# DSL-019: verify_invalid_block Epoch Mismatch Rejection

> **Authoritative requirement:** [DSL-019](../NORMATIVE.md#DSL-019)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

The `SlashingEvidence::epoch` envelope field MUST agree with `signed_header.message.epoch`. A mismatch indicates either a buggy reporter or a replay attempt and is rejected with `SlashingError::InvalidSlashingEvidence("epoch mismatch")`.

---

## Specification

```rust
if payload.signed_header.message.epoch != evidence.epoch {
    return Err(SlashingError::InvalidSlashingEvidence(
        format!("epoch mismatch: header={} envelope={}",
            payload.signed_header.message.epoch, evidence.epoch).into()));
}
```

---

## Acceptance Criteria

- [ ] Matching epochs → passes this check.
- [ ] Mismatched epochs → `InvalidSlashingEvidence` with descriptive message.
- [ ] Runs before BLS verify (cheap filter).
