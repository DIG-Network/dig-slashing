## Summary

`IndexedAttestation::validate_structure()` is a cheap, pre-signature structural guard. Rejects: non-ascending indices, duplicate indices, empty set, length > `MAX_VALIDATORS_PER_COMMITTEE` (2048), wrong signature byte-width. Returns `Ok(())` on well-formed structure. Callers run this BEFORE the expensive `aggregate_verify`.

---

## Specification

```rust
impl IndexedAttestation {
    pub fn validate_structure(&self) -> Result<(), SlashingError> {
        if self.attesting_indices.is_empty() {
            return Err(SlashingError::InvalidIndexedAttestation("empty indices".into()));
        }
        if self.attesting_indices.len() > MAX_VALIDATORS_PER_COMMITTEE {
            return Err(SlashingError::InvalidIndexedAttestation("indices len exceeds cap".into()));
        }
        if self.signature.len() != BLS_SIGNATURE_SIZE {
            return Err(SlashingError::InvalidIndexedAttestation("bad signature width".into()));
        }
        for w in self.attesting_indices.windows(2) {
            if w[0] >= w[1] {
                return Err(SlashingError::InvalidIndexedAttestation(
                    "non-ascending or duplicate index".into(),
                ));
            }
        }
        Ok(())
    }
}
```

---

## Acceptance Criteria

- [ ] Empty indices → `Err(InvalidIndexedAttestation(_))`.
- [ ] Indices over `MAX_VALIDATORS_PER_COMMITTEE` (2_048) → error.
- [ ] Non-ascending pair (`a >= b` for consecutive) → error.
- [ ] Duplicate (`a == b` case of above) → error.
- [ ] `signature.len() != 96` → error.
- [ ] Well-formed input returns `Ok(())`.
