# DSL-157: SlashingEvidence + Payload Serde Roundtrip

> **Authoritative:** [DSL-157](../NORMATIVE.md#DSL-157) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.5, §18

---

## Summary

`SlashingEvidence` and `SlashingEvidencePayload` MUST round-trip byte-exactly via `bincode` AND `serde_json`. Covers envelope + all three payload variants (`Proposer`, `Attester`, `InvalidBlock`). Required for REMARK wire (JSON) and internal storage (bincode).

---

## Specification

```rust
let encoded_bincode = bincode::serialize(&evidence).unwrap();
let decoded_bincode: SlashingEvidence = bincode::deserialize(&encoded_bincode).unwrap();
assert_eq!(evidence, decoded_bincode);

let encoded_json = serde_json::to_vec(&evidence).unwrap();
let decoded_json: SlashingEvidence = serde_json::from_slice(&encoded_json).unwrap();
assert_eq!(evidence, decoded_json);
```

---

## Acceptance Criteria

- [ ] ProposerSlashing payload round-trips via bincode + serde_json.
- [ ] AttesterSlashing payload round-trips via bincode + serde_json.
- [ ] InvalidBlockProof payload round-trips via bincode + serde_json.
- [ ] All envelope fields preserved byte-exact.
- [ ] `serde_bytes` fields (signatures, witness) encoded as raw bytes, not array-of-u8.
