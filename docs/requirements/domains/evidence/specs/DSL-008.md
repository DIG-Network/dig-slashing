## Summary

`InvalidBlockProof` carries one signed header + a caller-supplied failure witness + a categorical `InvalidBlockReason`. Eight reason variants. All fields survive serde round-trip byte-exactly (required for REMARK wire + `PendingSlash::evidence` storage).

---

## Specification

```rust
#[derive(Debug, Clone, Copy, Serialize, Deserialize, PartialEq, Eq, Hash)]
pub enum InvalidBlockReason {
    BadStateRoot,
    BadParentRoot,
    BadTimestamp,
    BadProposerIndex,
    TransactionExecutionFailure,
    OverweightBlock,
    DuplicateTransaction,
    Other,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct InvalidBlockProof {
    pub signed_header: SignedBlockHeader,
    #[serde(with = "serde_bytes")]
    pub failure_witness: Vec<u8>,
    pub failure_reason: InvalidBlockReason,
}
```

## Acceptance Criteria

- [ ] All 8 `InvalidBlockReason` variants enumerated.
- [ ] `bincode` round-trip preserves equality.
- [ ] `serde_json` round-trip preserves equality.
- [ ] `failure_witness` bytes serialize via `serde_bytes` (not array).
- [ ] `PartialEq` considers all three fields.
