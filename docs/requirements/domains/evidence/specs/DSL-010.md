## Summary

`SlashingEvidence::slashable_validators()` returns the list of validator indices the evidence accuses. For `Proposer` and `InvalidBlock`, the list has exactly one element (the `proposer_index` from the signed header). For `Attester`, the list is `AttesterSlashing::slashable_indices()` (the intersection, DSL-007) — zero or more.

Used by `verify_evidence` (DSL-012 reporter-self-accuse check) and by `SlashingManager::submit_evidence` to iterate the per-validator slash loop.

---

## Specification

```rust
impl SlashingEvidence {
    pub fn slashable_validators(&self) -> Vec<u32> {
        match &self.payload {
            SlashingEvidencePayload::Proposer(p) =>
                vec![p.signed_header_a.message.proposer_index],
            SlashingEvidencePayload::Attester(a) =>
                a.slashable_indices(),
            SlashingEvidencePayload::InvalidBlock(i) =>
                vec![i.signed_header.message.proposer_index],
        }
    }
}
```

---

## Acceptance Criteria

- [ ] Proposer payload → 1-element vec with `proposer_index`.
- [ ] InvalidBlock payload → 1-element vec with `proposer_index`.
- [ ] Attester payload with non-empty intersection → N-element vec, strictly ascending.
- [ ] Attester payload with empty intersection → empty vec.
- [ ] Deterministic.
