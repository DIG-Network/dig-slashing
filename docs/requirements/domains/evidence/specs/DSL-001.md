## Summary

`OffenseType::base_penalty_bps()` returns the BPS floor for the base-slash formula in `submit_evidence`. Four variants map to three distinct values: `ProposerEquivocation â†’ 500`, `InvalidBlock â†’ 300`, `AttesterDoubleVote â†’ 100`, `AttesterSurroundVote â†’ 100`. Every value stays under the `MAX_PENALTY_BPS` ceiling (1_000), which bounds the BPS term before correlation amplification is applied at finalisation.

BPS constants anchor the `max(bps_term, quotient_term)` formula from SPEC Â§4. Changing any one of these constants requires a protocol version bump.

## Specification

```rust
pub enum OffenseType {
    ProposerEquivocation,
    InvalidBlock,
    AttesterDoubleVote,
    AttesterSurroundVote,
}

impl OffenseType {
    pub fn base_penalty_bps(&self) -> u16 {
        match self {
            Self::ProposerEquivocation => EQUIVOCATION_BASE_BPS,   // 500
            Self::InvalidBlock         => INVALID_BLOCK_BASE_BPS,  // 300
            Self::AttesterDoubleVote
            | Self::AttesterSurroundVote => ATTESTATION_BASE_BPS,  // 100
        }
    }
}
```

Invariants:
- `base_penalty_bps() < MAX_PENALTY_BPS` for every variant.
- `base_penalty_bps() > 0` for every variant.
