## Summary

`AttestationData::signing_root(network_id)` produces the BLS signing message for attester slashings and attestation-participation verification. It is domain-prefixed with `DOMAIN_BEACON_ATTESTER` and network-bound by the network id. Any field mutation shifts the root.

---

## Specification

```rust
impl AttestationData {
    pub fn signing_root(&self, network_id: &Bytes32) -> Bytes32 {
        let mut h = Sha256::new();
        h.update(DOMAIN_BEACON_ATTESTER);
        h.update(network_id.as_ref());
        h.update(&self.slot.to_le_bytes());
        h.update(&self.index.to_le_bytes());
        h.update(self.beacon_block_root.as_ref());
        h.update(&self.source.epoch.to_le_bytes());
        h.update(self.source.root.as_ref());
        h.update(&self.target.epoch.to_le_bytes());
        h.update(self.target.root.as_ref());
        Bytes32::new(h.finalize().into())
    }
}
```

---

## Acceptance Criteria

- [ ] Deterministic: same input → same output.
- [ ] Prefixed with `DOMAIN_BEACON_ATTESTER` (`b"DIG_BEACON_ATTESTER_V1"`).
- [ ] Includes network_id (cross-network replay protection).
- [ ] Mutating any field (`slot`, `index`, `beacon_block_root`, `source.epoch`, `source.root`, `target.epoch`, `target.root`, `network_id`) changes the root.

---

## Implementation Notes

- **Primary file:** `src/evidence/attestation_data.rs`
- Uses `chia_sha2::Sha256`. Little-endian integer encoding throughout.
