## Summary

`verify_evidence` enforces the lookback window early. Evidence with `evidence.epoch + SLASH_LOOKBACK_EPOCHS < current_epoch` MUST return `SlashingError::OffenseTooOld { offense_epoch, current_epoch }`. Bounds memory (old evidence is unhashable after pruning) and prevents replay of stale evidence after honest validators have exited.

`SLASH_LOOKBACK_EPOCHS` is re-exported from `dig_epoch`; the constant is not redefined.

---

## Specification

```rust
pub fn verify_evidence(
    evidence: &SlashingEvidence,
    validator_view: &dyn ValidatorView,
    network_id: &Bytes32,
    current_epoch: u64,
) -> Result<VerifiedEvidence, SlashingError> {
    if evidence.epoch + dig_epoch::SLASH_LOOKBACK_EPOCHS < current_epoch {
        return Err(SlashingError::OffenseTooOld {
            offense_epoch: evidence.epoch,
            current_epoch,
        });
    }
    // ... further checks DSL-012+
}
```

---

## Acceptance Criteria

- [ ] Evidence at `current_epoch - SLASH_LOOKBACK_EPOCHS` exactly → accepted (boundary).
- [ ] Evidence one epoch earlier → `OffenseTooOld`.
- [ ] Error fields match the input values.
- [ ] No state mutation on rejection.
- [ ] Works for `current_epoch = 0` (no underflow; addition on LHS).
