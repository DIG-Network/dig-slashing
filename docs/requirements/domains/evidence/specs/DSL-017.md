## Summary

When neither the double-vote (DSL-014) nor surround-vote (DSL-015) predicate holds, the attestations are not slashable. `verify_attester_slashing` MUST return `SlashingError::AttesterSlashingNotSlashable`.

---

## Specification

```rust
let is_double_vote = a.data.target.epoch == b.data.target.epoch && a.data != b.data;
let is_surround_vote =
    (a.data.source.epoch < b.data.source.epoch && a.data.target.epoch > b.data.target.epoch) ||
    (b.data.source.epoch < a.data.source.epoch && b.data.target.epoch > a.data.target.epoch);
if !(is_double_vote || is_surround_vote) {
    return Err(SlashingError::AttesterSlashingNotSlashable);
}
```

---

## Acceptance Criteria

- [ ] Two attestations at completely different epoch windows with no surround → rejected.
- [ ] Same target epoch + same data → byte-identical → rejected earlier as `InvalidAttesterSlashing("identical")`, not this variant.
- [ ] Different target epoch + non-surrounding → rejected.
- [ ] One is proper double-vote → passes this check.
- [ ] One is proper surround-vote → passes this check.
