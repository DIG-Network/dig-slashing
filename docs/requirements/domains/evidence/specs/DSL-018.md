# DSL-018: verify_invalid_block Signature Over block_signing_message

## Summary

`verify_invalid_block` proves that the accused validator signed a block that is (claimed to be) invalid. The signature MUST verify over `dig_block::block_signing_message(network_id, header.epoch, &header.hash(), header.proposer_index)` — the same domain-bound message used for honest block production. Domain binding prevents cross-network replay.

---

## Specification

```rust
let msg = dig_block::block_signing_message(
    network_id,
    header.epoch,
    &header.hash(),
    header.proposer_index,
);
if !chia_bls::verify(&sig, &validator_pubkey, msg.as_ref()) {
    return Err(SlashingError::InvalidSlashingEvidence("bad invalid-block signature".into()));
}
```

---

## Acceptance Criteria

- [ ] Valid signature over `block_signing_message` → passes this check.
- [ ] Signature over raw `header.hash()` (no domain) → rejected (domain binding enforced).
- [ ] Signature from a different key → rejected.
- [ ] Signature over `block_signing_message` with different network_id → rejected.
- [ ] Uses `dig_block::block_signing_message` directly (no custom signing message).
- [ ] Uses `chia_bls::verify`.
