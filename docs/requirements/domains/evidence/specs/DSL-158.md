# DSL-158: IndexedAttestation Serde Roundtrip

> **Authoritative:** [DSL-158](../NORMATIVE.md#DSL-158) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.3, §18

---

## Summary

`IndexedAttestation` (nested `AttestationData` + `attesting_indices: Vec<u32>` + `signature: Vec<u8>`) MUST round-trip byte-exactly via `bincode` and `serde_json`. Required for `AttesterSlashing` carriage and mempool / REMARK wire.

---

## Specification

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub struct IndexedAttestation {
    pub attesting_indices: Vec<u32>,
    pub data: AttestationData,
    #[serde(with = "serde_bytes")]
    pub signature: Vec<u8>,
}
```

---

## Acceptance Criteria

- [ ] bincode round-trip equal.
- [ ] serde_json round-trip equal.
- [ ] `attesting_indices` order preserved.
- [ ] `signature` encoded via `serde_bytes` (not `[u8; N]`).
- [ ] Nested `AttestationData` (including `Checkpoint source/target`) preserved.
