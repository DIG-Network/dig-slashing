## Summary

`verify_proposer_slashing` proves that the accused validator signed two DIFFERENT block headers at the SAME slot — i.e. equivocation. Six preconditions: same-slot, same-proposer, different block roots, both signatures parse (96 bytes G2), validator is active at `header.epoch`, both signatures BLS-verify under the validator's pubkey against `dig_block::block_signing_message`.

Any precondition failure returns `SlashingError::InvalidProposerSlashing(reason)`.

---

## Specification

```rust
fn verify_proposer_slashing(
    evidence: &SlashingEvidence,
    payload: &ProposerSlashing,
    validator_view: &dyn ValidatorView,
    network_id: &Bytes32,
) -> Result<VerifiedEvidence, SlashingError>;
```

Preconditions (checked in order):
1. `header_a.slot == header_b.slot`
2. `header_a.proposer_index == header_b.proposer_index`
3. `header_a.hash() != header_b.hash()`
4. Both signatures parse as 96-byte G2 elements
5. Validator exists, is not already slashed, is active at `header.epoch`
6. `chia_bls::verify(sig, pk, block_signing_message(network_id, header.epoch, &header.hash(), header.proposer_index))` for both

---

## Acceptance Criteria

- [ ] Valid proposer slashing returns `Ok(VerifiedEvidence { slashable_validator_indices: [proposer_index] })`.
- [ ] Different slots → `InvalidProposerSlashing("slot mismatch")`.
- [ ] Different proposer indices → error.
- [ ] Identical header hashes → error.
- [ ] Malformed signature bytes → error.
- [ ] Validator missing or already slashed → `ValidatorNotRegistered` or skipped per policy.
- [ ] Validator inactive at `header.epoch` → error.
- [ ] BLS verify failure on sig_a or sig_b → error.
- [ ] Uses `dig_block::block_signing_message` (no custom signing message).
