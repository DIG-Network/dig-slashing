## Summary

At finalisation, each validator whose slash was not successfully appealed is hit with an **additional** correlation penalty. Unlike the base slash (DSL-022) which is applied optimistically at admission, the correlation penalty is deferred to window expiry ŌĆö its size depends on how much stake was slashed across the entire `CORRELATION_WINDOW_EPOCHS` window, which is only knowable once the window closes.

The formula (SPEC ┬¦4):

```
correlation_penalty = eff_bal
                    * min(cohort_sum * PROPORTIONAL_SLASHING_MULTIPLIER, total_active_balance)
                    / total_active_balance
```

where:

- `eff_bal` is the validator's current effective balance at finalisation (`EffectiveBalanceView::get(idx)`).
- `cohort_sum` is the sum of `eff_bal_at_slash` over every entry in `slashed_in_window` whose epoch lies in `[current_epoch - CORRELATION_WINDOW_EPOCHS, current_epoch]`.
- `PROPORTIONAL_SLASHING_MULTIPLIER == 3` (Ethereum parity).
- `total_active_balance` is passed in by the caller (consensus layer).

The `min(..., total_active_balance)` saturation caps the penalty at `eff_bal` (the maximum possible). The multiplier amplifies the cohort sum so that correlated attacks are punished more than independent ones.

The penalty is applied via `ValidatorEntry::slash_absolute(correlation_penalty, current_epoch)` and recorded in `FinalisationResult::per_validator_correlation_penalty`.

## Specification

### Formula

```rust
let window_lo = self.current_epoch.saturating_sub(CORRELATION_WINDOW_EPOCHS);
let cohort_sum: u64 = self.slashed_in_window
    .range((window_lo, 0)..=(self.current_epoch, u32::MAX))
    .map(|(_, eff_bal_at_slash)| *eff_bal_at_slash)
    .sum();

for entry in &pending.base_slash_per_validator {
    let idx = entry.validator_index;
    let eff_bal = effective_balances.get(idx);
    let scaled  = cohort_sum.saturating_mul(PROPORTIONAL_SLASHING_MULTIPLIER);
    let capped  = scaled.min(total_active_balance);
    let correlation_penalty = if total_active_balance == 0 {
        0
    } else {
        (eff_bal.saturating_mul(capped)) / total_active_balance
    };
    let _ = validator_set.get_mut(idx).unwrap()
        .slash_absolute(correlation_penalty, self.current_epoch);
    correlation_penalties.push((idx, correlation_penalty));
}
```

### Constants in scope

| Constant | Value | Source |
|----------|-------|--------|
| `CORRELATION_WINDOW_EPOCHS` | re-exported from `dig_epoch` | SPEC ┬¦2.2 |
| `PROPORTIONAL_SLASHING_MULTIPLIER` | `3` | SPEC ┬¦2.2 |

### Saturation Behaviour

| Scenario | `min(cohort_sum * 3, total)` | `correlation_penalty` |
|----------|------------------------------|-----------------------|
| Tiny cohort (isolated slash) | `cohort_sum * 3` | `eff_bal * 3 * cohort_sum / total` (small) |
| Large cohort (coordinated attack) | `total` (saturates) | `eff_bal` (entire balance) |
| `cohort_sum == 0` (impossible after dedup, defensive) | `0` | `0` |

### Result Population

`FinalisationResult::per_validator_correlation_penalty` MUST be a `Vec<(u32, u64)>` with one entry per validator in `pending.base_slash_per_validator`, preserving the same order.

## Acceptance Criteria

- [ ] Correlation penalty computed per validator using the formula above.
- [ ] `cohort_sum` covers only entries in `[current_epoch - CORRELATION_WINDOW_EPOCHS, current_epoch]`.
- [ ] `ValidatorEntry::slash_absolute` called with the computed penalty at `current_epoch`.
- [ ] `FinalisationResult::per_validator_correlation_penalty` populated in pending order.
- [ ] Saturation: when `cohort_sum * 3 >= total_active_balance`, penalty equals `eff_bal`.
- [ ] No overflow: arithmetic uses saturating multiplication where products could exceed `u64::MAX`.
- [ ] `total_active_balance == 0` produces zero penalty (defensive; not expected in practice).

## Implementation Notes

- **Primary file:** `src/manager.rs` + `src/lifecycle.rs`.
- **Reuses:** `EffectiveBalanceView::get`, `ValidatorEntry::slash_absolute`, `slashed_in_window` map populated by submit_evidence (DSL-022).
- **Edge cases:**
  - A single isolated slash ŌåÆ `cohort_sum = eff_bal_at_slash`; penalty is tiny (proportional to single-validator fraction of total).
  - A mass-slashing event where `cohort_sum * 3 >= total` ŌåÆ penalty saturates to `eff_bal`; effectively doubles the debit.
  - Validator's current `eff_bal` may differ from `eff_bal_at_slash` recorded at admission (e.g. if validator grew stake between admission and finalisation); formula uses the **current** balance.
  - If the validator is already exited at finalisation, `slash_absolute` still debits the remaining stake (per ┬¦15.1 contract).
