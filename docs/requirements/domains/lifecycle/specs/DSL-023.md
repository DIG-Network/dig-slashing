# DSL-023: submit_evidence Escrows Reporter Bond

> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Optimistic slashing is adversarial: a reporter submits evidence that immediately debits the accused validator's stake, but the slash can be reversed by a sustained appeal. To discourage frivolous or malicious reports, the reporter MUST post a bond of `REPORTER_BOND_MOJOS = MIN_EFFECTIVE_BALANCE / 64` at admission time. The bond is held in escrow for the duration of the appeal window (`SLASH_APPEAL_WINDOW_EPOCHS = 8`).

`SlashingManager::submit_evidence` delegates the escrow to the `BondEscrow` trait (see §12.3 and the `bonds/` domain, DSL-121..126). The manager calls `BondEscrow::lock(reporter_idx, REPORTER_BOND_MOJOS, BondTag::Reporter(evidence_hash))`. The escrow implementation holds the mojos against the reporter's stake; if the reporter's post-slash stake is below the bond amount the lock returns `InsufficientBalance`, which DSL-028 propagates as `BondLockFailed`.

The bond is refunded in full at finalisation (DSL-031) if no sustained appeal arrives, or forfeited on sustained appeal (see appeal domain DSL-068).

## Specification

### Call Site

```rust
// Inside SlashingManager::submit_evidence, after processed-check + verify_evidence,
// before any validator-set mutation:
let hash = evidence.hash();
let reporter_idx = evidence.reporter_validator_index;
bond_escrow
    .lock(reporter_idx, REPORTER_BOND_MOJOS, BondTag::Reporter(hash))
    .map_err(|_| SlashingError::BondLockFailed)?;
```

### Constants in scope

| Constant | Value |
|----------|-------|
| `REPORTER_BOND_MOJOS` | `MIN_EFFECTIVE_BALANCE / 64` (SPEC §2.6) |
| `BondTag::Reporter(Bytes32)` | variant of `BondTag` (SPEC §12.3) |

### Result Population

`SlashingResult::reporter_bond_escrowed` MUST equal `REPORTER_BOND_MOJOS` on success.

### Ordering Invariant

The bond lock MUST occur **before** `ValidatorEntry::slash_absolute` is called on any accused validator. If the lock fails, no validator state is mutated and no `PendingSlash` is inserted (this is shared with DSL-028).

---

## Acceptance Criteria

- [ ] `BondEscrow::lock` is called exactly once per successful `submit_evidence` invocation.
- [ ] Arguments are `(reporter_validator_index, REPORTER_BOND_MOJOS, BondTag::Reporter(evidence_hash))`.
- [ ] Lock occurs after `processed` dedup and `verify_evidence`, before validator slashes.
- [ ] `SlashingResult::reporter_bond_escrowed == REPORTER_BOND_MOJOS` on success.
- [ ] Lock failure surfaces as `SlashingError::BondLockFailed` with no state mutation (cross-referenced by DSL-028).
- [ ] `evidence_hash` used in the tag equals `evidence.hash()` (same value stored in `processed`).

---

## Implementation Notes

- **Primary file:** `src/manager.rs` (`SlashingManager::submit_evidence`).
- **Reuses:** `BondEscrow` trait (DSL-121..126 in `bonds/` domain).
- **Edge cases:**
  - A zero `REPORTER_BOND_MOJOS` (would require `MIN_EFFECTIVE_BALANCE < 64`) is not a realistic configuration; implementations MAY pass zero through unchanged.
  - Locking is a pure state mutation on the escrow; it never performs I/O.
