# DSL-147: PendingSlashBook::expired_by

> **Authoritative:** [DSL-147](../NORMATIVE.md#DSL-147) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §7.1

---

## Summary

`PendingSlashBook::expired_by(current_epoch)` returns the list of `evidence_hash` values whose `window_expires_at_epoch < current_epoch` AND whose status is `Accepted` or `ChallengeOpen` (i.e. not already Finalised/Reverted). Driven by the `by_window_expiry` BTreeMap index for efficient range query.

Consumed by `finalise_expired_slashes` (DSL-029).

---

## Specification

```rust
impl PendingSlashBook {
    pub fn expired_by(&self, current_epoch: u64) -> Vec<Bytes32>;
}
```

---

## Acceptance Criteria

- [ ] Entries with `window_expires_at_epoch < current_epoch` returned.
- [ ] Entries at exact boundary (`window_expires == current_epoch`) NOT returned (still in window).
- [ ] Reverted/Finalised entries excluded.
- [ ] Empty book → empty vec.
- [ ] Results ordered by expiry epoch ascending (deterministic).
