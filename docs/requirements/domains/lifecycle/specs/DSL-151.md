# DSL-151: Correlation Penalty Saturation Clamp

> **Authoritative:** [DSL-151](../NORMATIVE.md#DSL-151) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §4, §7.4

---

## Summary

Correlation penalty formula:
```
correlation = eff_bal * min(cohort_sum * PROPORTIONAL_SLASHING_MULTIPLIER, total_active_balance)
            / total_active_balance
```

The `min(..., total_active_balance)` clamp is non-negotiable: even when a mass-slash causes `cohort_sum * 3 > total_active_balance`, per-validator correlation penalty MUST NOT exceed `eff_bal` (full-stake ceiling).

---

## Specification

```rust
let cohort_sum: u64 = /* sum of eff_bal across in-window slashes */;
let numerator = std::cmp::min(
    cohort_sum.saturating_mul(PROPORTIONAL_SLASHING_MULTIPLIER),
    total_active_balance,
);
let correlation = (eff_bal as u128 * numerator as u128 / total_active_balance as u128) as u64;
```

---

## Acceptance Criteria

- [ ] Small cohort: `correlation == eff_bal * cohort_sum * 3 / total_active`.
- [ ] Mass cohort: clamp activates; `correlation <= eff_bal`.
- [ ] Exactly `cohort_sum * 3 == total_active` → `correlation == eff_bal`.
- [ ] Saturating multiplication prevents u64 overflow.
- [ ] `total_active == 0` → guard (formula undefined); return 0.
