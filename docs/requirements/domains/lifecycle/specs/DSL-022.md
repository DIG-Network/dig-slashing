# DSL-022: submit_evidence Applies Base Slash Formula

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

`SlashingManager::submit_evidence` is the entry point for **optimistic admission** of validator slashing evidence. For every validator named in `VerifiedEvidence::slashable_validator_indices`, the manager MUST compute the base slash amount as the larger of two terms:

1. A **BPS-proportional** term, `eff_bal * base_penalty_bps / BPS_DENOMINATOR`, parameterised by the `OffenseType` (500 bps for ProposerEquivocation, 300 for InvalidBlock, 100 for both attester variants).
2. An **Ethereum-parity floor**, `eff_bal / MIN_SLASHING_PENALTY_QUOTIENT` (i.e. `eff_bal / 32`), which guarantees a minimum burn even for low-bps offenses.

The chosen amount is debited from the validator's stake via `ValidatorEntry::slash_absolute(amount, current_epoch)`. This value is also stored in `PerValidatorSlash::base_slash_amount` and becomes the reversible "optimistic" portion of the slash — it is credited back on sustained appeal, remains applied on finalisation, and is joined by an additional correlation penalty only at window expiry.

The formula is pure integer arithmetic; there are no floating-point operations, no saturating casts, and no network I/O.

## Specification

### Function Context

```rust
impl SlashingManager {
    pub fn submit_evidence(
        &mut self,
        evidence: SlashingEvidence,
        validator_set: &mut dyn ValidatorView,
        effective_balances: &dyn EffectiveBalanceView,
        collateral: Option<&mut dyn CollateralSlasher>,
        bond_escrow: &mut dyn BondEscrow,
        reward_payout: &mut dyn RewardPayout,
        proposer: &dyn ProposerView,
        network_id: &Bytes32,
    ) -> Result<SlashingResult, SlashingError>;
}
```

### Formula

For each `idx` in `verified.slashable_validator_indices`:

```rust
let eff_bal      = effective_balances.get(idx);
let base_bps     = evidence.offense_type().base_penalty_bps() as u64;
let bps_term     = eff_bal * base_bps / BPS_DENOMINATOR;       // 10_000
let floor_term   = eff_bal / MIN_SLASHING_PENALTY_QUOTIENT;    // 32
let base_slash   = bps_term.max(floor_term);
let _ = validator_set.get_mut(idx).unwrap().slash_absolute(base_slash, current_epoch);
```

### Worked Examples

Given `MIN_EFFECTIVE_BALANCE = 32_000_000_000` mojos:

| Offense | `bps_term` | `floor_term` | `base_slash` |
|---------|-----------|--------------|--------------|
| ProposerEquivocation | `eff*500/10_000` = `eff/20` | `eff/32` | `eff/20` (bps wins) |
| InvalidBlock | `eff/33` | `eff/32` | `eff/32` (floor wins by one mojo) |
| AttesterDoubleVote | `eff/100` | `eff/32` | `eff/32` (floor wins) |
| AttesterSurroundVote | `eff/100` | `eff/32` | `eff/32` (floor wins) |

## Acceptance Criteria

- [ ] The formula `base_slash = max(eff_bal * base_bps / 10_000, eff_bal / 32)` is applied unchanged per validator.
- [ ] `ValidatorEntry::slash_absolute(base_slash, current_epoch)` is invoked exactly once per slashable validator.
- [ ] `PerValidatorSlash::base_slash_amount` carries the computed value.
- [ ] `PerValidatorSlash::effective_balance_at_slash` equals `EffectiveBalanceView::get(idx)` captured at submission.
- [ ] Validators already flagged `is_slashed()` are skipped (no second debit).
- [ ] Arithmetic is exclusively integer; no intermediate overflow for `u64` values within valid effective-balance range.

---

## Implementation Notes

- **Primary file:** `src/manager.rs` (`SlashingManager::submit_evidence`), with helpers in `src/lifecycle.rs`.
- **Reuses:** `ValidatorView` + `ValidatorEntry::slash_absolute` (DSL-121..126 bond domain provides `BondEscrow`); `EffectiveBalanceView`.
- **Edge cases:**
  - A slashable index missing from `validator_set` MUST be skipped silently (defensive; `verify_evidence` is authoritative, but the manager tolerates drift).
  - `eff_bal == 0` yields `base_slash == 0`; still emits `PerValidatorSlash` with zeros.
  - Overflow: `eff_bal * 500` fits in `u64` so long as `eff_bal < u64::MAX / 500`; `MIN_EFFECTIVE_BALANCE` is far below that ceiling.
