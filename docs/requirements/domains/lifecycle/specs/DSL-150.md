# DSL-150: SlashingManager::is_processed + pending + prune

> **Authoritative:** [DSL-150](../NORMATIVE.md#DSL-150) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §7.2

---

## Summary

Three manager query / maintenance methods:

- `is_processed(hash)` reads the `processed` dedup map.
- `pending(hash)` returns `Option<&PendingSlash>`.
- `prune(before_epoch)` drops `processed` entries with stored epoch `< before_epoch` and `slashed_in_window` entries with slash epoch `< before_epoch`.

Typical caller: at epoch boundary, `prune(current_epoch - SLASH_LOOKBACK_EPOCHS - 1)`.

---

## Specification

```rust
impl SlashingManager {
    pub fn is_processed(&self, h: &Bytes32) -> bool;
    pub fn pending(&self, h: &Bytes32) -> Option<&PendingSlash>;
    pub fn prune(&mut self, before_epoch: u64);
}
```

---

## Acceptance Criteria

- [ ] `is_processed(known_hash)` → true.
- [ ] `is_processed(unknown_hash)` → false.
- [ ] `pending(hash)` → Some for pending slashes, None otherwise.
- [ ] `prune(before)` removes `processed` entries with `stored_epoch < before`.
- [ ] `prune(before)` removes `slashed_in_window` entries with `slash_epoch < before`.
- [ ] `prune` does NOT touch `book` (pending slashes removed via `remove` or `finalise`).
