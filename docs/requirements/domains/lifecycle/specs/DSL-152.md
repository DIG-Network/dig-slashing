# DSL-152: submit_evidence Reporter-Is-Accused Short-Circuit

> **Authoritative:** [DSL-152](../NORMATIVE.md#DSL-152) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §5.1, §7.3

---

## Summary

`submit_evidence` MUST propagate the `SlashingError::ReporterIsAccused(idx)` check from `verify_evidence` (DSL-012) BEFORE any bond lock, reward payout, state mutation, or PendingSlash insertion. Ensures a malicious reporter cannot even attempt to accuse themselves.

---

## Specification

Ordering at the top of `submit_evidence`:
1. `verify_evidence` — returns `ReporterIsAccused` if reporter in slashable set.
2. Any downstream state mutation only if verify succeeds.

---

## Acceptance Criteria

- [ ] Self-accuse evidence → ReporterIsAccused; no bond locked; no reward paid; no PendingSlash inserted; `processed` unchanged.
- [ ] Distinct reporter → proceeds normally.
- [ ] Attester-slashing with reporter in intersection → ReporterIsAccused (reporter is ONE of the accused validators).
