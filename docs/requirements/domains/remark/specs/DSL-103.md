# DSL-103: Evidence Puzzle Reveal Emits One REMARK

> **Authoritative:** [DSL-103](../NORMATIVE.md#DSL-103) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.2, §22.12

---

## Summary

`slashing_evidence_remark_puzzle_reveal_v1` returns a CLVM program that, when executed via `chia_sdk_types::run_puzzle`, emits exactly ONE `Condition::Remark { message }` carrying the encoded payload. Puzzle hash via `clvm_utils::tree_hash`.

---

## Specification

```rust
pub fn slashing_evidence_remark_puzzle_reveal_v1(ev: &SlashingEvidence) -> Result<Vec<u8>, SpendError>;
pub fn slashing_evidence_remark_puzzle_hash_v1(ev: &SlashingEvidence) -> Result<Bytes32, SpendError>;
```

The puzzle is a quoted `(1 REMARK <payload>)` construct; its tree hash is the coin's `puzzle_hash`.

---

## Acceptance Criteria

- [ ] `run_puzzle(reveal, solution=[])` yields exactly one Remark condition.
- [ ] Remark message parses back to the original evidence.
- [ ] `puzzle_hash` stable across invocations.
- [ ] `puzzle_hash` computed via `clvm_utils::tree_hash`.
