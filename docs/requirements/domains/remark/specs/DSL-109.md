# DSL-109: Evidence Payload Cap

> **Authoritative:** [DSL-109](../NORMATIVE.md#DSL-109) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.3, §2.8, §22.12

---

## Summary

Evidence whose serialized JSON payload exceeds `MAX_SLASH_PROPOSAL_PAYLOAD_BYTES` (65_536) MUST be rejected via `SlashingRemarkError::PayloadTooLarge`.

---

## Specification

```rust
let len = serde_json::to_vec(ev)?.len();
if len > MAX_SLASH_PROPOSAL_PAYLOAD_BYTES {
    return Err(SlashingRemarkError::PayloadTooLarge {
        actual: len, limit: MAX_SLASH_PROPOSAL_PAYLOAD_BYTES,
    });
}
```

---

## Acceptance Criteria

- [ ] `> 65_536` bytes → PayloadTooLarge.
- [ ] At limit → ok.
- [ ] Small payload → ok.
