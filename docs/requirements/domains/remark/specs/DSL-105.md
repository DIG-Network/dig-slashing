# DSL-105: Evidence Admission Mismatch Rejected

> **Authoritative:** [DSL-105](../NORMATIVE.md#DSL-105) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.2, §22.12

---

## Summary

When the spent coin's `puzzle_hash` differs from `slashing_evidence_remark_puzzle_hash_v1(&evidence)`, admission MUST return `SlashingRemarkError::AdmissionPuzzleHashMismatch { got, expected }` with both hashes hex-encoded.

---

## Specification

```rust
if spend.coin.puzzle_hash != expected {
    return Err(SlashingRemarkError::AdmissionPuzzleHashMismatch {
        got: hex::encode(spend.coin.puzzle_hash.as_ref()),
        expected: hex::encode(expected.as_ref()),
    });
}
```

---

## Acceptance Criteria

- [ ] Mismatched puzzle_hash → AdmissionPuzzleHashMismatch.
- [ ] Both hashes hex-encoded in error.
- [ ] Other spends in the same bundle are not considered after first mismatch.
