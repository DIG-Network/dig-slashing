# DSL-102: Evidence REMARK Wire Roundtrip

> **Authoritative:** [DSL-102](../NORMATIVE.md#DSL-102) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.1, §16.2, §22.12

---

## Summary

`encode_slashing_evidence_remark_payload_v1` produces `SLASH_EVIDENCE_REMARK_MAGIC_V1 || serde_json::to_vec(evidence)`. `parse_slashing_evidence_from_conditions` strips the magic prefix and deserialises. Roundtrip is byte-exact.

---

## Specification

```rust
pub fn encode_slashing_evidence_remark_payload_v1(ev: &SlashingEvidence) -> Result<Vec<u8>, _> {
    let mut out = SLASH_EVIDENCE_REMARK_MAGIC_V1.to_vec();
    out.extend_from_slice(&serde_json::to_vec(ev)?);
    Ok(out)
}
```

---

## Acceptance Criteria

- [ ] Encoded payload begins with magic prefix.
- [ ] Decoded message equals original evidence.
- [ ] Non-magic prefix → skipped by parser.
- [ ] Malformed JSON after magic → skipped.
