# DSL-107: Evidence Mempool Policy — Duplicate Rejected

> **Authoritative:** [DSL-107](../NORMATIVE.md#DSL-107) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.3, §22.12

---

## Summary

`enforce_slashing_evidence_mempool_policy` MUST reject evidence that either (a) matches an entry already in `pending_evidence` OR (b) appears multiple times in `incoming_evidence`. Fingerprint = JSON bytes.

---

## Specification

```rust
let mut seen = HashSet::<Vec<u8>>::new();
for ev in pending_evidence { seen.insert(serde_json::to_vec(ev)?); }
for ev in incoming_evidence {
    let fp = serde_json::to_vec(ev)?;
    if !seen.insert(fp) {
        return Err(SlashingRemarkError::DuplicateEvidence);
    }
}
```

---

## Acceptance Criteria

- [ ] Dup vs pending → DuplicateEvidence.
- [ ] Dup within incoming → DuplicateEvidence.
- [ ] Distinct evidence → ok.
