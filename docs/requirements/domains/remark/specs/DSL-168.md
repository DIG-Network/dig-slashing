# DSL-168: process_block_admissions Block-Level Entry Point

> **Authoritative:** [DSL-168](../NORMATIVE.md#DSL-168) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.5, §7.3

---

## Summary

Add a single public entry point `process_block_admissions(conditions, manager, vs, balances, bond, reward, proposer, network_id, current_epoch) -> BlockAdmissionReport` that:

1. Parses slashing-evidence REMARK conditions via `parse_slashing_evidence_from_conditions` (DSL-106).
2. Parses slash-appeal REMARK conditions via `parse_slash_appeals_from_conditions` (DSL-111).
3. Enforces block-level caps (`enforce_block_level_slashing_caps` DSL-119 + `enforce_block_level_appeal_caps`).
4. Enforces per-payload admission policy (`enforce_slashing_evidence_remark_admission` + `enforce_slash_appeal_remark_admission`).
5. Calls `SlashingManager::submit_evidence` for each admitted evidence, then `submit_appeal` for each admitted appeal.
6. Collects per-envelope outcomes into a single `BlockAdmissionReport`.

Current state: embedder must iterate REMARKs and call each parse → policy → submit by hand. This requirement composes the six-step pipeline into one call.

## Specification

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, Default)]
pub struct BlockAdmissionReport {
    /// SlashingResult per successfully-admitted evidence envelope,
    /// paired with its evidence_hash.
    pub admitted_evidences: Vec<(Bytes32, SlashingResult)>,
    /// (evidence_hash, SlashingError) for each rejected evidence.
    pub rejected_evidences: Vec<(Bytes32, SlashingError)>,
    /// appeal_hash for each admitted appeal.
    pub admitted_appeals: Vec<Bytes32>,
    /// (appeal_hash, SlashingError) for each rejected appeal.
    pub rejected_appeals: Vec<(Bytes32, SlashingError)>,
    /// Count of envelopes dropped by block-cap enforcement.
    pub cap_dropped_evidences: usize,
    pub cap_dropped_appeals: usize,
}

#[allow(clippy::too_many_arguments)]
pub fn process_block_admissions(
    conditions: &[Condition],
    manager: &mut SlashingManager,
    validator_set: &mut dyn ValidatorView,
    effective_balances: &dyn EffectiveBalanceView,
    bond_escrow: &mut dyn BondEscrow,
    reward_payout: &mut dyn RewardPayout,
    proposer: &dyn ProposerView,
    network_id: &Bytes32,
    current_epoch: u64,
) -> BlockAdmissionReport;
```

### Ordering (normative)

1. Evidence REMARKs parsed + admission-policy + cap-enforced BEFORE appeal REMARKs, so appeals can reference evidence admitted in the same block.
2. Each evidence envelope: `submit_evidence` failures are recorded in `rejected_evidences` without aborting the block.
3. Each appeal envelope: `submit_appeal` failures are recorded in `rejected_appeals` without aborting the block.
4. Cap-dropped envelopes (block exceeds `MAX_SLASH_PROPOSALS_PER_BLOCK` / `MAX_APPEALS_PER_BLOCK`) are counted but NOT submitted.

---

## Acceptance Criteria

- [ ] Empty conditions slice → empty report, zero mutations.
- [ ] Mixed valid + invalid evidences → valid admitted into book; invalid surface in `rejected_evidences`.
- [ ] Appeal referencing evidence admitted earlier in the same block succeeds (evidence processed first).
- [ ] Cap-exceeding evidences dropped + counted in `cap_dropped_evidences` (not submitted).
- [ ] Cap-exceeding appeals dropped + counted in `cap_dropped_appeals`.
- [ ] Duplicate evidence within a block: first admits, second surfaces `AlreadySlashed` in rejected list.
- [ ] No panic on malformed REMARK payload — parse failure short-circuits per envelope.
- [ ] Report serde roundtrips via bincode + serde_json.

## Implementation Notes

- **Primary file:** `src/remark/admission.rs` (new module) OR `src/remark/mod.rs` extension.
- Does NOT touch `SlashingManager::submit_evidence` / `submit_appeal` signatures.
- Composes existing DSL-106/111/119/120/022..024/062 public surface.
- Export `BlockAdmissionReport` + `process_block_admissions` from `src/remark/mod.rs` + `src/lib.rs`.
- `Condition` type comes from `chia_protocol` (the REMARK condition enum).
