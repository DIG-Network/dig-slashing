# DSL-114: Appeal Mempool AppealForUnknownSlash

> **Authoritative:** [DSL-114](../NORMATIVE.md#DSL-114) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §16.6, §22.13

---

## Summary

`enforce_slash_appeal_mempool_policy` MUST reject an appeal whose `evidence_hash` is NOT in the pending-slash set via `SlashingRemarkError::AppealForUnknownSlash(hex_hash)`.

---

## Specification

```rust
if !pending_slashes.contains(&appeal.evidence_hash) {
    return Err(SlashingRemarkError::AppealForUnknownSlash(
        hex::encode(appeal.evidence_hash.as_ref())
    ));
}
```

## Acceptance Criteria

- [ ] Known pending hash → ok.
