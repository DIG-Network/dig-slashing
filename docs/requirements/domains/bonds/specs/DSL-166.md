# DSL-166: BondTag::Reporter vs Appellant Distinguishable

> **Authoritative:** [DSL-166](../NORMATIVE.md#DSL-166) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §12.3

---

## Summary

`BondTag::Reporter(evidence_hash)` and `BondTag::Appellant(appeal_hash)` MUST be distinguishable even when their inner `Bytes32` values are bit-equal (hypothetical case — evidence and appeal hashes use different domain prefixes so collisions are cryptographically implausible, but the enum discriminant MUST still differ).

`escrowed(idx, Reporter(x))` and `escrowed(idx, Appellant(x))` MUST index separate escrow slots.

---

## Specification

```rust
#[derive(Debug, Clone, Copy, Serialize, Deserialize, PartialEq, Eq, Hash)]
pub enum BondTag {
    Reporter(Bytes32),
    Appellant(Bytes32),
}
```

---

## Acceptance Criteria

- [ ] `BondTag::Reporter(h)` != `BondTag::Appellant(h)` via `PartialEq` even when `h` identical.
- [ ] `Hash` output differs between the two variants with identical inner bytes.
- [ ] `BondEscrow::lock(idx, amount, Reporter(h))` followed by `escrowed(idx, Appellant(h))` returns 0.
- [ ] Serde round-trips preserve variant discriminator.
- [ ] `Copy` derive works (both variants).
