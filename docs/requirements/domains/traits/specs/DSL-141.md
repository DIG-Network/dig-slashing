# DSL-141: RewardPayout::pay

> **Authoritative:** [DSL-141](../NORMATIVE.md#DSL-141) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §12.1

---

## Summary

`RewardPayout::pay(principal_ph, amount)` credits a pay-to-puzzle-hash reward account. `dig-slashing` calls this for whistleblower rewards, proposer rewards, bond-split awards. Implementation lives downstream — may batch/coalesce.

Zero-amount calls are no-ops.

---

## Specification

```rust
pub trait RewardPayout {
    fn pay(&mut self, principal_ph: Bytes32, amount: u64);
}
```

---

## Acceptance Criteria

- [ ] Single pay recorded against `principal_ph` with `amount`.
- [ ] Multiple pays to same address accumulate (additive in total payout).
- [ ] `amount == 0` → no-op (no-op at minimum; may still record a zero entry).
