# DSL-140: BondEscrow::escrowed

> **Authoritative:** [DSL-140](../NORMATIVE.md#DSL-140) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §12.3, §15.4

---

## Summary

`BondEscrow::escrowed(principal_idx, tag)` is the read-side query. Returns the mojos currently escrowed under `(principal_idx, tag)`, or 0 for an unknown pairing. Never panics. Used by adjudicator/lifecycle/tests to inspect escrow state without lock/forfeit/release mutation.

```rust
pub trait BondEscrow {
    fn escrowed(&self, principal_idx: u32, tag: BondTag) -> u64;
    // ... lock / release / forfeit in DSL-121/122/123
}
```

---

## Acceptance Criteria

- [ ] After `lock(idx, amount, tag)` → `escrowed(idx, tag) == amount`.
- [ ] After `release` or `forfeit` → `escrowed(idx, tag) == 0`.
- [ ] Unknown `(idx, tag)` → 0 (no panic).
- [ ] Read-only (no internal mutation).
