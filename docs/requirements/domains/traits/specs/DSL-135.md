# DSL-135: ValidatorEntry::schedule_exit

> **Authoritative:** [DSL-135](../NORMATIVE.md#DSL-135) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.1

## Summary

`schedule_exit(exit_lock_until_epoch)` persists the post-slash exit lock. After `finalise_expired_slashes` (DSL-032), the slashed validator's effective `exit_epoch` becomes `current_epoch + SLASH_LOCK_EPOCHS` (100).

---

## Specification

```rust
fn schedule_exit(&mut self, exit_lock_until_epoch: u64);
```

---

## Acceptance Criteria

- [ ] After call, `exit_epoch()` returns `exit_lock_until_epoch`.
- [ ] Overwritable (last call wins).
- [ ] Downstream `is_active_at_epoch` observes the new exit epoch.
