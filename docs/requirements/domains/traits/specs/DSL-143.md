# DSL-143: JustificationView Contract

> **Authoritative:** [DSL-143](../NORMATIVE.md#DSL-143) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.3

---

## Summary

`JustificationView` is the read-side interface to fork-choice state that `dig-slashing` consults during appeal verification (notably DSL-075 source-justified, DSL-076 target-canonical). Five methods, all read-only.

---

## Specification

```rust
pub trait JustificationView {
    fn current_justified_checkpoint(&self) -> Checkpoint;
    fn previous_justified_checkpoint(&self) -> Checkpoint;
    fn finalized_checkpoint(&self) -> Checkpoint;
    fn canonical_block_root_at_slot(&self, slot: u64) -> Option<Bytes32>;
    fn canonical_target_root_for_epoch(&self, epoch: u64) -> Option<Bytes32>;
}
```

---

## Acceptance Criteria

- [ ] Every method callable without error.
- [ ] `canonical_block_root_at_slot` returns `None` for uncommitted/future slots.
- [ ] `canonical_target_root_for_epoch` returns `None` for epochs past chain tip.
- [ ] Checkpoints returned have valid `(epoch, root)` pairs.
- [ ] Read-only — no trait method mutates.
