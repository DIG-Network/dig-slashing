# DSL-139: CollateralSlasher slash+credit Symmetry

> **Authoritative:** [DSL-139](../NORMATIVE.md#DSL-139) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.2

---

## Summary

`CollateralSlasher::slash(idx, amount, epoch)` debits validator collateral; `credit(idx, amount)` returns it. Round-trip restores prior state. Missing-collateral (`NoCollateral`) is a soft failure at `submit_evidence` (DSL-022) — slash succeeds on stake even if no collateral.

---

## Specification

```rust
pub trait CollateralSlasher {
    fn slash(&mut self, idx: u32, amount: u64, epoch: u64) -> Result<(u64, u64), CollateralError>;
    fn credit(&mut self, idx: u32, amount: u64) -> Result<u64, CollateralError>;
}
```

---

## Acceptance Criteria

- [ ] slash returns `(slashed, remaining)` on success.
- [ ] credit after slash restores collateral.
- [ ] No-collateral slash returns `Err(NoCollateral)`; caller ignores.
- [ ] Double-credit idempotent if nothing was slashed.
