# DSL-132: ValidatorEntry::credit_stake

> **Authoritative:** [DSL-132](../NORMATIVE.md#DSL-132) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.1

---

## Summary

`credit_stake(amount_mojos)` is the inverse of `slash_absolute` — adds mojos back to stake. Used by sustained-appeal adjudication (DSL-064) and reorg rewind (DSL-129). Returns mojos credited (always equals input unless overflow at `u64::MAX`).

---

## Specification

```rust
pub trait ValidatorEntry {
    fn credit_stake(&mut self, amount_mojos: u64) -> u64;
}
```

Semantics: `stake = stake.saturating_add(amount_mojos); return amount_mojos` (or saturated delta on overflow).

---

## Acceptance Criteria

- [ ] `stake += amount_mojos`.
- [ ] Returns credited amount.
- [ ] Inverse of slash_absolute: `slash(x)` then `credit(x)` restores original stake.
- [ ] Saturates at u64::MAX (no overflow panic).
