# DSL-144: ProposerView::proposer_at_slot

> **Authoritative:** [DSL-144](../NORMATIVE.md#DSL-144) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.3

---

## Summary

`ProposerView::proposer_at_slot(slot)` returns `Some(validator_index)` for committed slots with an assigned proposer, `None` otherwise (future slot, missed-slot, or uncommitted). `current_slot()` returns the current L2 slot.

Used by `submit_evidence` (DSL-025) to route proposer inclusion reward, and by `run_epoch_boundary` to query stall state against slots.

---

## Specification

```rust
pub trait ProposerView {
    fn proposer_at_slot(&self, slot: u64) -> Option<u32>;
    fn current_slot(&self) -> u64;
}
```

---

## Acceptance Criteria

- [ ] Committed slot with assigned proposer → `Some(idx)`.
- [ ] Future slot (beyond `current_slot`) → `None`.
- [ ] Missed-slot (no proposer) → `None`.
- [ ] `current_slot()` monotonically advances as chain progresses.
