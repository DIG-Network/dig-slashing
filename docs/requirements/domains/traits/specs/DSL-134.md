# DSL-134: ValidatorEntry::is_active_at_epoch Boundary

> **Authoritative:** [DSL-134](../NORMATIVE.md#DSL-134) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §15.1

---

## Summary

`is_active_at_epoch(epoch)` returns `true` iff `activation_epoch <= epoch < exit_epoch`. Left-inclusive, right-exclusive. Matches Ethereum validator-lifecycle semantics.

---

## Specification

```rust
fn is_active_at_epoch(&self, epoch: u64) -> bool {
    self.activation_epoch() <= epoch && epoch < self.exit_epoch()
}
```

---

## Acceptance Criteria

- [ ] `epoch == activation_epoch` → true.
- [ ] `epoch == activation_epoch - 1` → false (pre-activation).
- [ ] `epoch == exit_epoch - 1` → true.
- [ ] `epoch == exit_epoch` → false.
