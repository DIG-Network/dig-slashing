# DSL-093: InactivityScoreTracker::resize_for

> **Authoritative:** [DSL-093](../NORMATIVE.md#DSL-093) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §9.2, §10, §22.10

---

## Summary

`resize_for(validator_count)` grows or shrinks the `scores` vector. New slots initialise to 0. Shrinking truncates trailing scores — only used on exit-heavy epochs (unusual in practice; callers typically only grow).

Called by `run_epoch_boundary` step 7 when validator count changes (new activations).

---

## Specification

```rust
pub fn resize_for(&mut self, validator_count: usize) {
    self.scores.resize(validator_count, 0);
}
```

---

## Acceptance Criteria

- [ ] Growing from N to N+K → K new entries, each 0.
- [ ] Existing entries unchanged.
- [ ] Shrinking → trailing entries dropped.
- [ ] No-op when size matches.
