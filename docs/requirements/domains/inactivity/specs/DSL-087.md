# DSL-087: in_finality_stall Threshold

> **Authoritative:** [DSL-087](../NORMATIVE.md#DSL-087) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §9.1, §2.4, §22.10

---

## Summary

`in_finality_stall(current_epoch, finalized_epoch)` returns `true` iff `current_epoch - finalized_epoch > MIN_EPOCHS_TO_INACTIVITY_PENALTY` (= 4). Strict greater-than — at exactly 4 epochs gap, not yet in stall. Saturating subtraction handles `current < finalized` edge (post-reorg transient).

---

## Specification

```rust
pub fn in_finality_stall(current_epoch: u64, finalized_epoch: u64) -> bool {
    current_epoch.saturating_sub(finalized_epoch) > MIN_EPOCHS_TO_INACTIVITY_PENALTY
}
```

---

## Acceptance Criteria

- [ ] `current - finalized > 4` → true.
- [ ] `current - finalized == 4` → false (strict greater-than).
- [ ] `current - finalized == 0` → false.
- [ ] `current < finalized` (saturating) → false.
