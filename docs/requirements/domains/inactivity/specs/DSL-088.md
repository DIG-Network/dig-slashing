# DSL-088: InactivityScoreTracker Hit Decrement

> **Authoritative:** [DSL-088](../NORMATIVE.md#DSL-088) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §9.2, §22.10

---

## Summary

For every validator that HIT `TIMELY_TARGET` in the previous epoch, `update_for_epoch` decrements their inactivity score by 1 (saturating at 0). Applies whether or not the chain is in stall — timely target participation always rewards a score decrement.

---

## Specification

```rust
for (i, flags) in previous_epoch_participation.iter().enumerate() {
    if flags.is_target_timely() {
        self.scores[i] = self.scores[i].saturating_sub(1);
    }
    // miss branch in DSL-089
}
```

---

## Acceptance Criteria

- [ ] TIMELY_TARGET set → score -= 1 (saturating).
- [ ] score = 0 stays 0 (saturating).
- [ ] Hit in stall → still decrements.
- [ ] Hit out of stall → still decrements (global recovery in DSL-090 is additional).
