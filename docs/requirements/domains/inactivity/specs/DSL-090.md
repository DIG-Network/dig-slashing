# DSL-090: InactivityScoreTracker Out-of-Stall Recovery

> **Authoritative:** [DSL-090](../NORMATIVE.md#DSL-090) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §9.2, §2.4, §22.10

---

## Summary

When NOT in a finality stall, `update_for_epoch` decreases every validator's inactivity score by `INACTIVITY_SCORE_RECOVERY_RATE` (= 16), saturating at 0. Runs AFTER the per-validator hit/miss update.

Matches Ethereum Bellatrix global recovery semantics — scores decay quickly once finality is restored.

---

## Specification

```rust
if !in_finality_stall {
    for score in &mut self.scores {
        *score = score.saturating_sub(INACTIVITY_SCORE_RECOVERY_RATE);
    }
}
```

---

## Acceptance Criteria

- [ ] out-of-stall → every score decreases by 16.
- [ ] score = 10 → 0 (saturating, not negative).
- [ ] In-stall → no global recovery (only per-validator hit decrement from DSL-088).
- [ ] Applies after per-validator updates (DSL-088/089).
