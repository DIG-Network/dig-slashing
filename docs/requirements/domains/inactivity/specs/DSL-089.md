# DSL-089: InactivityScoreTracker Miss + Stall Increment

> **Authoritative:** [DSL-089](../NORMATIVE.md#DSL-089) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §9.2, §2.4, §22.10

---

## Summary

For validators that MISSED `TIMELY_TARGET` in the previous epoch WHILE the chain is in a finality stall, `update_for_epoch` increments their score by `INACTIVITY_SCORE_BIAS` (= 4). Saturating at `u64::MAX`. Outside the stall, missed targets do NOT increment.

---

## Specification

```rust
if !flags.is_target_timely() && in_finality_stall {
    self.scores[i] = self.scores[i].saturating_add(INACTIVITY_SCORE_BIAS);
}
```

---

## Acceptance Criteria

- [ ] Miss + stall → score += 4.
- [ ] Miss + no stall → no increment (DSL-090 applies global recovery).
- [ ] score at u64::MAX stays (saturating).
- [ ] Multiple consecutive misses in stall → score grows linearly.
