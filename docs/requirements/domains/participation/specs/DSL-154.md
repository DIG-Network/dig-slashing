# DSL-154: ParticipationFlags Serde Roundtrip

> **Authoritative:** [DSL-154](../NORMATIVE.md#DSL-154) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.10, §18

---

## Summary

`ParticipationFlags(u8)` MUST round-trip byte-exactly via `bincode` and `serde_json`. Required for `EpochBoundaryReport` + `ReorgReport` serialisation and for on-disk snapshotting by consumers.

---

## Specification

```rust
#[derive(Debug, Clone, Copy, Default, Serialize, Deserialize, PartialEq, Eq, Hash)]
pub struct ParticipationFlags(pub u8);
```

---

## Acceptance Criteria

- [ ] `bincode` roundtrip preserves byte value.
- [ ] `serde_json` roundtrip preserves byte value.
- [ ] Vec<ParticipationFlags> roundtrips.
- [ ] Each bit pattern (0..=7) roundtrips.
