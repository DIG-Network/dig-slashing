# DSL-074: ParticipationFlags Bit Set/Has

> **Authoritative:** [DSL-074](../NORMATIVE.md#DSL-074) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.10, §2.9, §22.9

---

## Summary

`ParticipationFlags(u8)` is a bitmask storing the three Ethereum Altair flags: bit 0 = `TIMELY_SOURCE`, bit 1 = `TIMELY_TARGET`, bit 2 = `TIMELY_HEAD`. `set(flag_index)` sets the bit additively; `has(flag_index)` reads.

---

## Specification

```rust
#[derive(Debug, Clone, Copy, Default, Serialize, Deserialize, PartialEq, Eq, Hash)]
pub struct ParticipationFlags(pub u8);

impl ParticipationFlags {
    pub fn set(&mut self, flag_index: u8) { self.0 |= 1 << flag_index; }
    pub fn has(&self, flag_index: u8) -> bool { (self.0 >> flag_index) & 1 == 1 }
    pub fn is_source_timely(&self) -> bool { self.has(TIMELY_SOURCE_FLAG_INDEX) }
    pub fn is_target_timely(&self) -> bool { self.has(TIMELY_TARGET_FLAG_INDEX) }
    pub fn is_head_timely(&self)   -> bool { self.has(TIMELY_HEAD_FLAG_INDEX) }
}
```

---

## Acceptance Criteria

- [ ] `set(0)` sets bit 0; other bits unchanged.
- [ ] `has(0)` returns true iff bit 0 set.
- [ ] Setting a bit twice is idempotent.
- [ ] Convenience accessors (`is_source_timely`, etc.) match `has` semantics.
- [ ] Default is `ParticipationFlags(0)`.
