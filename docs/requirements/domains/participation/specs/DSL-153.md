# DSL-153: ParticipationTracker::rewind_on_reorg

> **Authoritative:** [DSL-153](../NORMATIVE.md#DSL-153) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.2, §13

---

## Summary

`ParticipationTracker::rewind_on_reorg(depth, validator_count)` restores tracker state from its internal snapshot ring-buffer (depth of `CORRELATION_WINDOW_EPOCHS`). Drops the last `depth` epochs of recorded flags. Participation flags become those captured at `current_epoch - depth`.

Called by `rewind_all_on_reorg` (DSL-130) during fork-choice reorg.

---

## Specification

```rust
impl ParticipationTracker {
    pub fn rewind_on_reorg(&mut self, depth: u64, validator_count: usize);
}
```

On reorg deeper than the snapshot window, `rewind_all_on_reorg` returns `ReorgTooDeep` (DSL-130) and this method is never invoked.

---

## Acceptance Criteria

- [ ] Tracker state after rewind matches snapshot at `current_epoch - depth`.
- [ ] `current_epoch` and `previous_epoch` arrays restored from ring-buffer.
- [ ] `current_epoch_number` decremented by `depth`.
- [ ] `validator_count` resize applied (shrink or grow to match new active set).
- [ ] No-op on `depth == 0`.

---

## Implementation Notes

- **Primary file:** `src/participation/tracker.rs`
- Snapshot ring buffer depth = `CORRELATION_WINDOW_EPOCHS` (36).
