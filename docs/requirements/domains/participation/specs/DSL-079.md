# DSL-079: ParticipationTracker Non-Ascending Rejection

> **Authoritative:** [DSL-079](../NORMATIVE.md#DSL-079) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.2, §22.9

---

## Summary

`record_attestation` MUST reject non-ascending indices (`NonAscendingIndices`) and duplicates (`DuplicateIndex`). Mirrors `IndexedAttestation::validate_structure` (DSL-005).

---

## Specification

```rust
for w in attesting_indices.windows(2) {
    if w[0] == w[1] {
        return Err(ParticipationError::DuplicateIndex(w[0]));
    }
    if w[0] > w[1] {
        return Err(ParticipationError::NonAscendingIndices);
    }
}
```

---

## Acceptance Criteria

- [ ] `[3, 2, 1]` → NonAscendingIndices.
- [ ] `[1, 1, 2]` → DuplicateIndex(1).
- [ ] `[1]` → ok (single-element).
- [ ] `[1, 2, 3]` → ok.
