# DSL-077: classify_timeliness TIMELY_HEAD

> **Authoritative:** [DSL-077](../NORMATIVE.md#DSL-077) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.1, §2.5, §22.9

---

## Summary

Sets `TIMELY_HEAD` iff `delay == MIN_ATTESTATION_INCLUSION_DELAY` (= 1) AND `head_is_canonical` is true. Strictest timing — matches Ethereum head-vote inclusion rule.

---

## Specification

```rust
if delay == MIN_ATTESTATION_INCLUSION_DELAY
   && head_is_canonical {
    flags.set(TIMELY_HEAD_FLAG_INDEX);
}
```

---

## Acceptance Criteria

- [ ] delay=1 + canonical head → HEAD set.
- [ ] delay=2 → NOT set (outside one-slot window).
- [ ] delay=0 → NOT set.
- [ ] Non-canonical head → NOT set.
