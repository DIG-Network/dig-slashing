# DSL-084: compute_flag_deltas Stall Zeroes Rewards

> **Authoritative:** [DSL-084](../NORMATIVE.md#DSL-084) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.3, §22.9

---

## Summary

In a finality stall (`in_finality_stall == true`), all rewards are zeroed. Penalties still apply. Mirrors Ethereum leak semantics: while finality is stalled, rewards are suppressed but missed-attestation penalties continue to debit.

---

## Specification

```rust
if in_finality_stall {
    reward = 0;
}
// penalty remains from DSL-083
```

---

## Acceptance Criteria

- [ ] in_finality_stall=true → reward=0 for every validator.
- [ ] in_finality_stall=true → penalties still applied per DSL-083.
- [ ] in_finality_stall=false → rewards applied per DSL-082 normally.
