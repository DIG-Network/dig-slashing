# DSL-078: ParticipationTracker::record_attestation

> **Authoritative:** [DSL-078](../NORMATIVE.md#DSL-078) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.2, §22.9

---

## Summary

`ParticipationTracker::record_attestation(data, attesting_indices, flags)` applies `flags` to each validator in `attesting_indices` via bit-OR into `current_epoch[idx]`. Additive: subsequent inclusion of the same validator adds more flag bits, never overwrites.

---

## Specification

```rust
pub fn record_attestation(
    &mut self,
    data: &AttestationData,
    attesting_indices: &[u32],
    flags: ParticipationFlags,
) -> Result<(), ParticipationError> {
    // DSL-079 structural checks first
    for idx in attesting_indices {
        let i = *idx as usize;
        if i >= self.current_epoch.len() {
            return Err(ParticipationError::IndexOutOfRange(*idx));
        }
        self.current_epoch[i].0 |= flags.0;
    }
    Ok(())
}
```

---

## Acceptance Criteria

- [ ] Flags applied to every `attesting_indices` entry.
- [ ] Bit-OR (additive), not overwrite.
- [ ] Out-of-range index → IndexOutOfRange.
- [ ] Subsequent record with different flags adds bits.
- [ ] `current_epoch_number` unchanged.
