# DSL-081: base_reward Formula

> **Authoritative:** [DSL-081](../NORMATIVE.md#DSL-081) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.3, §2.3, §22.9

---

## Summary

Ethereum Altair base reward formula. Per-validator, per-epoch reward basis that scales INVERSELY with the integer sqrt of total active stake. Smaller networks pay more per validator; larger networks dilute.

```
base_reward = effective_balance * BASE_REWARD_FACTOR / integer_sqrt(total_active_balance)
```

---

## Specification

```rust
use num_integer::Roots;

pub fn base_reward(effective_balance: u64, total_active_balance: u64) -> u64 {
    let denom = (total_active_balance as u128).sqrt() as u64;
    if denom == 0 { return 0; }
    (effective_balance as u128)
        .saturating_mul(BASE_REWARD_FACTOR as u128)
        .saturating_div(denom as u128) as u64
}
```

---

## Acceptance Criteria

- [ ] Formula matches `eff_bal * 64 / isqrt(total_active)` for representative inputs.
- [ ] Uses `num_integer::Roots::sqrt` (not a custom isqrt).
- [ ] Saturates at u64::MAX on overflow.
- [ ] `total_active_balance = 0` → returns 0 (guard).
- [ ] Larger `total_active_balance` → smaller base_reward (inverse-sqrt).
