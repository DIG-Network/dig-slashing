# DSL-080: ParticipationTracker::rotate_epoch

> **Authoritative:** [DSL-080](../NORMATIVE.md#DSL-080) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §8.2, §10, §22.9

---

## Summary

`rotate_epoch(new_epoch, validator_count)` is the epoch-boundary transition. Swaps `previous_epoch` and `current_epoch`, then clears and resizes `current_epoch` to `validator_count` of zero-initialised flags. Updates `current_epoch_number`.

Called by `run_epoch_boundary` step 5 (SPEC §10).

---

## Specification

```rust
pub fn rotate_epoch(&mut self, new_epoch: u64, validator_count: usize) {
    std::mem::swap(&mut self.previous_epoch, &mut self.current_epoch);
    self.current_epoch.clear();
    self.current_epoch.resize(validator_count, ParticipationFlags::default());
    self.current_epoch_number = new_epoch;
}
```

## Acceptance Criteria

- [ ] `current_epoch_number` updated.
- [ ] Post-rotation: `previous_epoch` holds what was `current` before.
