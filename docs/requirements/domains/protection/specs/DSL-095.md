# DSL-095: check_attestation Same-Epoch Different-Hash Fails

> **Authoritative:** [DSL-095](../NORMATIVE.md#DSL-095) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.1, §22.11

---

## Summary

At the same `(source_epoch, target_epoch)` pair, the validator may only re-sign the SAME block hash. Attempting to attest a different hash at the same FFG coordinates fails — that would be a self-double-vote.

---

## Specification

```rust
if source_epoch == self.last_attested_source_epoch
   && target_epoch == self.last_attested_target_epoch {
    let candidate_hex = format!("0x{}", hex::encode(block_hash.as_ref()));
    if self.last_attested_block_hash.as_deref() != Some(candidate_hex.as_str()) {
        return false;
    }
}
```

---

## Acceptance Criteria

- [ ] Same (src, tgt), same hash → true.
- [ ] Same (src, tgt), different hash → false.
- [ ] Hex comparison case-insensitive.
- [ ] `None` stored hash at same (src, tgt) → false (cannot confirm match).
