# DSL-101: Save/Load Roundtrip

> **Authoritative:** [DSL-101](../NORMATIVE.md#DSL-101) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.4, §22.11

---

## Summary

`SlashingProtection::save(path)` followed by `SlashingProtection::load(path)` MUST preserve every field byte-exactly. `last_attested_block_hash` is encoded as `0x<lowercase_hex>` and compared case-insensitively on load.

---

## Specification

```rust
impl SlashingProtection {
    pub fn save(&self, path: &PathBuf) -> std::io::Result<()> {
        let json = serde_json::to_vec_pretty(self)?;
        std::fs::write(path, json)
    }
    pub fn load(path: &PathBuf) -> std::io::Result<Self> {
        if !path.exists() { return Ok(Self::default()); }
        let bytes = std::fs::read(path)?;
        Ok(serde_json::from_slice(&bytes)?)
    }
}
```

---

## Acceptance Criteria

- [ ] Roundtrip preserves all six fields.
- [ ] Hash encoded lowercase hex with `0x` prefix.
- [ ] `load` on non-existent path returns default.
- [ ] Case-insensitive hash compare on reload (validator tooling may write uppercase).
