# DSL-156: SlashingProtection::rewind_proposal_to_slot

> **Authoritative:** [DSL-156](../NORMATIVE.md#DSL-156) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.3

---

## Summary

`rewind_proposal_to_slot(new_tip_slot)` lowers `last_proposed_slot` to at most `new_tip_slot`. Companion to `rewind_attestation_to_epoch` (DSL-098). Called by `reconcile_with_chain_tip` (DSL-099) and `rewind_all_on_reorg` (DSL-130).

If `last_proposed_slot <= new_tip_slot` (no rewind needed), the method is a no-op.

---

## Specification

```rust
impl SlashingProtection {
    pub fn rewind_proposal_to_slot(&mut self, new_tip_slot: u64) {
        if self.last_proposed_slot > new_tip_slot {
            self.last_proposed_slot = new_tip_slot;
        }
    }
}
```

---

## Acceptance Criteria

- [ ] `last_proposed_slot > new_tip_slot` → lowered to `new_tip_slot`.
- [ ] `last_proposed_slot <= new_tip_slot` → unchanged.
- [ ] Idempotent.
