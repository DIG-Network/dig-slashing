# DSL-098: rewind_attestation_to_epoch Clears Hash

> **Authoritative:** [DSL-098](../NORMATIVE.md#DSL-098) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.3, §22.11

---

## Summary

`rewind_attestation_to_epoch(new_tip_epoch)` is called on fork-choice reorg or chain-tip refresh. Clears `last_attested_block_hash` to `None` and lowers `last_attested_source_epoch` and `last_attested_target_epoch` to values no greater than `new_tip_epoch`. Lets the validator re-attest on the canonical tip after reorg.

---

## Specification

```rust
pub fn rewind_attestation_to_epoch(&mut self, new_tip_epoch: u64) {
    if self.last_attested_source_epoch > new_tip_epoch {
        self.last_attested_source_epoch = new_tip_epoch;
    }
    if self.last_attested_target_epoch > new_tip_epoch {
        self.last_attested_target_epoch = new_tip_epoch;
    }
    self.last_attested_block_hash = None;
}
```

## Acceptance Criteria

- [ ] `last_attested_block_hash` cleared to None.
- [ ] Source and target epochs lowered to at most `new_tip_epoch`.
- [ ] Already-lower epochs unchanged.
- [ ] Subsequent re-attestation on canonical tip succeeds.
