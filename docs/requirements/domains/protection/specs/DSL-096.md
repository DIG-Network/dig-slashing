# DSL-096: would_surround Self-Check

> **Authoritative:** [DSL-096](../NORMATIVE.md#DSL-096) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.2, §22.11

---

## Summary

`check_attestation` consults `would_surround` before signing. Returns `true` if the candidate attestation would surround the previously-attested one:
```
candidate_source < last_attested_source_epoch
AND candidate_target > last_attested_target_epoch
```
Signing a surround-vote is self-slashable; `check_attestation` rejects it to protect the validator.

---

## Specification

```rust
fn would_surround(&self, candidate_source: u64, candidate_target: u64) -> bool {
    candidate_source < self.last_attested_source_epoch
    && candidate_target > self.last_attested_target_epoch
}

pub fn check_attestation(&self, source: u64, target: u64, hash: &Bytes32) -> bool {
    if self.would_surround(source, target) { return false; }
    // ... DSL-095 same-epoch check
    true
}
```

---

## Acceptance Criteria

- [ ] Candidate strictly surrounds prior → false.
- [ ] Candidate matches prior (src, tgt) → check DSL-095 instead.
- [ ] Candidate flanks (strictly later than prior) → ok.
- [ ] Same source, higher target → not a surround (not < on source).
- [ ] Lower source, same target → not a surround (not > on target).
