# DSL-097: record_proposal / record_attestation Persist

> **Authoritative:** [DSL-097](../NORMATIVE.md#DSL-097) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.1, §22.11

---

## Summary

`record_proposal(slot)` sets `last_proposed_slot`. `record_attestation(source, target, hash)` sets `last_attested_source_epoch`, `last_attested_target_epoch`, and `last_attested_block_hash = Some(0x<hex>)`. Values persist across `save` / `load`.

---

## Specification

```rust
pub fn record_proposal(&mut self, slot: u64) { self.last_proposed_slot = slot; }

pub fn record_attestation(&mut self, source: u64, target: u64, hash: &Bytes32) {
    self.last_attested_source_epoch = source;
    self.last_attested_target_epoch = target;
    self.last_attested_block_hash = Some(format!("0x{}", hex::encode(hash.as_ref())));
}
```

---

## Acceptance Criteria

- [ ] `record_proposal` updates slot field.
- [ ] `record_attestation` updates all three fields.
- [ ] Hash encoded as `0x<lowercase_hex>`.
- [ ] Values survive `save` + `load`.
