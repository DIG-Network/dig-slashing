# DSL-099: reconcile_with_chain_tip Rewinds Both

> **Authoritative:** [DSL-099](../NORMATIVE.md#DSL-099) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §14.3, §22.11

---

## Summary

`reconcile_with_chain_tip(tip_slot, tip_epoch)` rewinds BOTH the proposal watermark (to `tip_slot`) AND the attestation watermark (to `tip_epoch`). Called on validator startup and after reorg.

---

## Specification

```rust
pub fn reconcile_with_chain_tip(&mut self, tip_slot: u64, tip_epoch: u64) {
    self.rewind_proposal_to_slot(tip_slot);
    self.rewind_attestation_to_epoch(tip_epoch);
}
```

---

## Acceptance Criteria

- [ ] Both watermarks lowered.
- [ ] `last_attested_block_hash` cleared to None.
- [ ] Idempotent (second call with same inputs is a no-op).
- [ ] Surround-vote self-check (DSL-096) re-passes after reconcile.
