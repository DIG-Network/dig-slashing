# DSL-159: SlashAppeal::hash Determinism + Sensitivity

> **Authoritative:** [DSL-159](../NORMATIVE.md#DSL-159) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.7, §2.10

---

## Summary

`SlashAppeal::hash()` is the content-addressed identity of an appeal. Used by `pending.appeal_history` dedup (DSL-058) and `AppealAttempt::appeal_hash` recording (DSL-070, DSL-072). SHA-256 over `DOMAIN_SLASH_APPEAL || bincode::serialize(appeal)`. Deterministic across runs; any field mutation shifts the hash.

---

## Specification

```rust
impl SlashAppeal {
    pub fn hash(&self) -> Bytes32 {
        let mut h = Sha256::new();
        h.update(DOMAIN_SLASH_APPEAL);
        h.update(&bincode::serialize(self).unwrap());
        Bytes32::new(h.finalize().into())
    }
}
```

---

## Acceptance Criteria

- [ ] Same appeal → same `Bytes32` across multiple calls.
- [ ] Mutating `evidence_hash`, `appellant_index`, `appellant_puzzle_hash`, `filed_epoch`, OR any byte of `payload` changes the hash.
- [ ] Domain-prefixed with `DOMAIN_SLASH_APPEAL`.
- [ ] Uses `chia_sha2::Sha256`.
