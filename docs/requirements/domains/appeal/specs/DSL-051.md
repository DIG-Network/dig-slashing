# DSL-051: InvalidBlockAppeal FailureReasonMismatch Sustained

> **Authoritative:** [DSL-051](../NORMATIVE.md#DSL-051) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.4, §22.6

---

## Summary

Sustains when the oracle's re-execution reports an `InvalidBlockReason` different from the `evidence.failure_reason` claimed in the original `InvalidBlockProof`. Block WAS invalid, but not for the reason the reporter claimed — either reporter bug or mis-classification.

---

## Specification

```rust
let oracle = invalid_block_oracle.ok_or(AppealError::MissingOracle("FailureReasonMismatch"))?;
match oracle.re_execute(&header, &witness)? {
    ExecutionOutcome::Invalid(actual) if actual != evidence.failure_reason =>
        AppealVerdict::Sustained { reason: AppealSustainReason::FailureReasonMismatch },
    _ => AppealVerdict::Rejected { .. },
}
```

---

## Acceptance Criteria

- [ ] Oracle returns `Invalid(different_reason)` → Sustained.
- [ ] Oracle returns `Invalid(claimed_reason)` → Rejected.
- [ ] Oracle returns `Valid` → Rejected (use DSL-049 ground instead).
- [ ] No oracle → `MissingOracle`.
