# DSL-171: AppealVerdict::to_appeal_outcome Conversion

> **Authoritative:** [DSL-171](../NORMATIVE.md#DSL-171) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** ┬¦3.8, ┬¦6.5

---

## Summary

`AppealVerdict` (producer side ŌĆö `Sustained { reason: AppealSustainReason }` / `Rejected { reason: AppealRejectReason }`) and `AppealOutcome` (recorded side ŌĆö `Won` / `Lost { reason_hash } ` / `Pending`) are presently bridged implicitly inside each adjudicator slice. Add an explicit conversion `AppealVerdict::to_appeal_outcome() -> AppealOutcome` so the DSL-167 dispatcher (and any future consumers) have ONE canonical mapping.

Conversion:

- `Sustained { .. }` ŌåÆ `Won`.
- `Rejected { reason }` ŌåÆ `Lost { reason_hash: SHA-256(bincode(reason)) }` ŌĆö deterministic, domain-agnostic (the raw enum discriminant + any payload fields contribute to the hash).

`Pending` is never produced by `to_appeal_outcome` ŌĆö it's a transient state used inside `AppealAttempt` BEFORE adjudication runs; the conversion output is always terminal.

---

## Specification

```rust
impl AppealVerdict {
    /// Maps the producer-side verdict into the recorded-side
    /// outcome carried on AppealAttempt + AppealAdjudicationResult.
    ///
    /// DSL-171. The hash for Rejected cases is deterministic under
    /// `chia_sha2::Sha256(bincode::serialize(reason))`.
    #[must_use]
    pub fn to_appeal_outcome(&self) -> AppealOutcome {
        match self {
            AppealVerdict::Sustained { .. } => AppealOutcome::Won,
            AppealVerdict::Rejected { reason } => {
                let mut h = chia_sha2::Sha256::new();
                let encoded = bincode::serialize(reason)
                    .expect("AppealRejectReason bincode must not fail");
                h.update(&encoded);
                let out: [u8; 32] = h.finalize();
                AppealOutcome::Lost {
                    reason_hash: Bytes32::new(out),
                }
            }
        }
    }
}
```

## Acceptance Criteria

- [ ] `Sustained { reason: HeadersIdentical }` ŌåÆ `Won`.
- [ ] `Rejected { reason: GroundDoesNotHold }` ŌåÆ `Lost { reason_hash }` where `reason_hash == SHA-256(bincode(GroundDoesNotHold))`.
- [ ] Conversion is deterministic across calls (same input ŌåÆ same output).
- [ ] Different `AppealRejectReason` variants produce DIFFERENT `reason_hash` values.
- [ ] Conversion NEVER yields `AppealOutcome::Pending`.
- [ ] `#[must_use]` on the method so callers can't accidentally drop the outcome.
