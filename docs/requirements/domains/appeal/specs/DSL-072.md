# DSL-072: Adjudicate Rejected Keeps ChallengeOpen + appeal_count+1

> **Authoritative:** [DSL-072](../NORMATIVE.md#DSL-072) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.5, §22.8

---

## Summary

On rejected appeal, `pending.status` MUST transition to (or remain) `ChallengeOpen { first_appeal_filed_epoch, appeal_count: n + 1 }` and `AppealAttempt { outcome: Lost { reason_hash } }` appended to history. Slash persists; further appeals accepted up to MAX_APPEAL_ATTEMPTS_PER_SLASH.

---

## Specification

```rust
let new_count = match pending.status {
    PendingSlashStatus::Accepted =>
        PendingSlashStatus::ChallengeOpen {
            first_appeal_filed_epoch: appeal.filed_epoch,
            appeal_count: 1,
        },
    PendingSlashStatus::ChallengeOpen { first_appeal_filed_epoch, appeal_count } =>
        PendingSlashStatus::ChallengeOpen {
            first_appeal_filed_epoch, appeal_count: appeal_count + 1,
        },
    _ => unreachable!(),
};
pending.status = new_count;
pending.appeal_history.push(AppealAttempt {
    outcome: AppealOutcome::Lost { reason_hash: ... }, ..
});
```

---

## Acceptance Criteria

- [ ] First rejected: status Accepted → ChallengeOpen { count: 1 }.
- [ ] Nth rejected: count increments.
- [ ] `first_appeal_filed_epoch` preserved after first rejection.
- [ ] AppealAttempt { Lost } appended.
