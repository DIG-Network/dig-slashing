# DSL-050: InvalidBlockAppeal ProposerSignatureInvalid Sustained

> **Authoritative:** [DSL-050](../NORMATIVE.md#DSL-050) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.4, §22.6

---

## Summary

Sustains when the signed header's BLS signature does NOT verify over `dig_block::block_signing_message`. No oracle needed — pure BLS check.

---

## Specification

```rust
let msg = dig_block::block_signing_message(
    network_id, header.epoch, &header.hash(), header.proposer_index);
if !chia_bls::verify(&sig, &pk, msg.as_ref()) {
    return AppealVerdict::Sustained { reason: AppealSustainReason::ProposerSignatureInvalid };
}
```

---

## Acceptance Criteria

- [ ] Corrupted signature → Sustained.
- [ ] Valid signature → Rejected.
- [ ] Sign under different network_id → Sustained.
- [ ] Uses `dig_block::block_signing_message` + `chia_bls::verify`.
