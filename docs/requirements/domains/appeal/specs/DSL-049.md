# DSL-049: InvalidBlockAppeal BlockActuallyValid Sustained

> **Authoritative:** [DSL-049](../NORMATIVE.md#DSL-049) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.4, §15.3, §22.6

---

## Summary

Sustains when `InvalidBlockOracle::re_execute(header, witness)` returns `ExecutionOutcome::Valid`. The oracle re-runs block validation with the full block body + pre-state + parent witness supplied in the appeal. If validation succeeds, the original evidence was incorrect and the slash is reverted.

Without an oracle: returns `AppealError::MissingOracle("BlockActuallyValid requires InvalidBlockOracle")`.

---

## Specification

```rust
let oracle = invalid_block_oracle.ok_or(AppealError::MissingOracle("BlockActuallyValid"))?;
match oracle.re_execute(&signed_header.message, &appeal.witness)? {
    ExecutionOutcome::Valid =>
        AppealVerdict::Sustained { reason: AppealSustainReason::BlockActuallyValid },
    ExecutionOutcome::Invalid(_) =>
        AppealVerdict::Rejected { reason: AppealRejectReason::OracleAgreesInvalid },
}
```

---

## Acceptance Criteria

- [ ] Oracle returns `Valid` → Sustained.
- [ ] Oracle returns `Invalid(_)` → Rejected.
- [ ] Oracle absent → `AppealError::MissingOracle`.
- [ ] Witness passed through to oracle unchanged.
