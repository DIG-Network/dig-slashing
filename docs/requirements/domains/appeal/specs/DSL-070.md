# DSL-070: Adjudicate Sustained Status Reverted

> **Authoritative:** [DSL-070](../NORMATIVE.md#DSL-070) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.5, §22.8

---

## Summary

After all sustained-adjudication steps, the adjudicator MUST set `pending.status = PendingSlashStatus::Reverted { winning_appeal_hash, reverted_at_epoch: current_epoch }` and append `AppealAttempt { outcome: Won }` to `pending.appeal_history`.

Subsequent `submit_appeal` calls referencing this evidence hash return `SlashAlreadyReverted` (DSL-060).

---

## Specification

```rust
pending.status = PendingSlashStatus::Reverted {
    winning_appeal_hash: appeal.hash(),
    reverted_at_epoch: current_epoch,
};
pending.appeal_history.push(AppealAttempt {
    appeal_hash: appeal.hash(),
    appellant_index: appeal.appellant_index,
    filed_epoch: appeal.filed_epoch,
    outcome: AppealOutcome::Won,
    bond_mojos: APPELLANT_BOND_MOJOS,
});
```

---

## Acceptance Criteria

- [ ] Status transitions to `Reverted { .. }`.
- [ ] `winning_appeal_hash` == `appeal.hash()`.
- [ ] `reverted_at_epoch` == `current_epoch`.
- [ ] `AppealAttempt { outcome: Won }` appended to history.
- [ ] Subsequent submit_appeal returns SlashAlreadyReverted.
