# DSL-047: Attester Appeal ValidatorNotInIntersection Sustained

> **Authoritative:** [DSL-047](../NORMATIVE.md#DSL-047) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.3, §22.5

---

## Summary

Sustains when a specific validator index (carried in `ground.ValidatorNotInIntersection { validator_index }`) is NOT in the intersection of the two attestations. Used to rescue a falsely-included validator from a buggy verifier admission.

---

## Specification

```rust
match ground {
    AttesterAppealGround::ValidatorNotInIntersection { validator_index } => {
        let slashable = evidence.slashable_indices();
        if !slashable.contains(&validator_index) {
            return AppealVerdict::Sustained {
                reason: AppealSustainReason::ValidatorNotInIntersection
            };
        }
    }
    _ => {}
}
```

---

## Acceptance Criteria

- [ ] Named index NOT in intersection → Sustained.
- [ ] Named index IS in intersection → Rejected.
- [ ] Only the named index is affected; other indices retain their slash state.
- [ ] Sustained verdict triggers per-validator revert only for the named index (adjudicator may restore a subset).

---

## Implementation Notes

- **Primary file:** `src/appeal/verify.rs`
- Adjudication note: when sustained, `AppealAdjudicationResult::reverted_stake_mojos` should include only this index — not all originally slashed.
