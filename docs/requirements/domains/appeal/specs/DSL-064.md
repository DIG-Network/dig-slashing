# DSL-064: Adjudicate Sustained Reverts Base Slash via credit_stake

> **Authoritative:** [DSL-064](../NORMATIVE.md#DSL-064) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.5, §22.8

---

## Summary

On `AppealVerdict::Sustained`, the adjudicator MUST credit each `PerValidatorSlash::base_slash_amount` back to the validator's stake via `ValidatorEntry::credit_stake(amount)`.

For `ValidatorNotInIntersection` ground (DSL-047), only the named index is credited; other indices remain slashed. For all other sustained grounds, every index in `pending.base_slash_per_validator` is credited.

---

## Specification

```rust
let indices_to_revert = match verdict {
    AppealVerdict::Sustained {
        reason: AppealSustainReason::ValidatorNotInIntersection
    } => vec![appeal.payload.named_index()],
    AppealVerdict::Sustained { .. } =>
        pending.base_slash_per_validator.iter().map(|s| s.validator_index).collect(),
    _ => vec![],
};
for idx in indices_to_revert {
    let slash = pending.base_slash_per_validator.iter().find(|s| s.validator_index == idx)?;
    validator_set.get_mut(idx).unwrap().credit_stake(slash.base_slash_amount);
}
```

---

## Acceptance Criteria

- [ ] `credit_stake(amount, current_epoch)` called per slashed validator.
- [ ] Amount equals `base_slash_amount`.
- [ ] For multi-validator sustained appeals, all indices restored.
- [ ] For `ValidatorNotInIntersection`, only named index restored.
