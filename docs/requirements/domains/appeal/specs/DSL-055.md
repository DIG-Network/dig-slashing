# DSL-055: submit_appeal UnknownEvidence

> **Authoritative:** [DSL-055](../NORMATIVE.md#DSL-055) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.1, §7.2, §22.7

---

## Summary

`submit_appeal` MUST reject `AppealError::UnknownEvidence(hash)` when `appeal.evidence_hash` is not found in `PendingSlashBook`. Bond is NOT locked on this failure path.

---

## Specification

```rust
let pending = self.book.get(&appeal.evidence_hash)
    .ok_or(AppealError::UnknownEvidence(hex::encode(appeal.evidence_hash)))?;
```

Ordering: check BEFORE any bond lock.

---

## Acceptance Criteria

- [ ] Fresh hash (no matching PendingSlash) → `UnknownEvidence`.
- [ ] Known hash → passes through to verification.
- [ ] Bond NOT locked on this error.
- [ ] Error carries hex-encoded hash.
