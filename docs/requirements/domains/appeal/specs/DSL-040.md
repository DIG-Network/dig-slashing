# DSL-040: Proposer Appeal Rejected on False Claim

## Summary

A `ProposerSlashingAppeal` whose ground does NOT hold on the evidence MUST be rejected. Covers all six grounds: `HeadersIdentical` on distinct headers, `ProposerIndexMismatch` on matching indices, `SignatureAInvalid`/`SignatureBInvalid` on genuine signatures, `SlotMismatch` on same slots, `ValidatorNotActiveAtEpoch` on active validators.

Rejected appeals forfeit the appellant bond per DSL-071.

---

## Specification

Every sustaining condition is a positive predicate. Failing to match the predicate yields `AppealVerdict::Rejected { reason: AppealRejectReason::GroundDoesNotHold }`.

---

## Acceptance Criteria

- [ ] `HeadersIdentical` ground, distinct headers → Rejected.
- [ ] `ProposerIndexMismatch` ground, matching indices → Rejected.
- [ ] `SignatureAInvalid` ground, genuine sig_a → Rejected.
- [ ] `SignatureBInvalid` ground, genuine sig_b → Rejected.
- [ ] `SlotMismatch` ground, same slot → Rejected.
- [ ] `ValidatorNotActiveAtEpoch` ground, active validator → Rejected.
