## Summary

Sustains the appeal when `header_a.proposer_index != header_b.proposer_index` — a buggy verifier admitted evidence naming a proposer who didn't sign BOTH headers.

---

## Specification

```rust
if evidence.signed_header_a.message.proposer_index
   != evidence.signed_header_b.message.proposer_index {
    return AppealVerdict::Sustained { reason: AppealSustainReason::ProposerIndexMismatch };
}
```

---

## Acceptance Criteria

- [ ] Different proposer indices → Sustained.
- [ ] Matching proposer indices → Rejected.
- [ ] Off-by-one proposer index → Sustained.
