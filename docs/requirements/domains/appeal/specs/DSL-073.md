# DSL-073: Adjudicate Clawback Shortfall Absorbed from Bond

> **Authoritative:** [DSL-073](../NORMATIVE.md#DSL-073) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.5, §22.8

---

## Summary

If the reporter has already withdrawn their optimistic `wb_reward` before the appeal is sustained, `RewardClawback::claw_back` may return less than the owed amount. The shortfall MUST be absorbed from the forfeited reporter bond (burn leg). If bond insufficient, residue flagged in telemetry (but adjudication succeeds).

---

## Specification

```rust
let clawback_shortfall = (wb_amount + prop_amount) - (got_wb + got_prop);
// DSL-068: appellant_award = forfeited_bond * 50%; burn = forfeited - appellant_award
// DSL-073: burn += clawback_shortfall (may exceed original burn share if shortfall large)
let final_burn = burn + clawback_shortfall;
// If final_burn > forfeited, residue = final_burn - forfeited; log warning.
```

---

## Acceptance Criteria

- [ ] `clawback_shortfall` populated in `AppealAdjudicationResult`.
- [ ] Shortfall absorbed into burn accounting.
- [ ] Bond sufficient → no residue.
- [ ] Bond insufficient → residue flagged via `tracing::warn!`, adjudication proceeds.
- [ ] Zero shortfall → no change from DSL-068 math.
