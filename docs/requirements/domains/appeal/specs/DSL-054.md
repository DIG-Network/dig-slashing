# DSL-054: InvalidBlockAppeal Rejected on Genuine Invalid Block

> **Authoritative:** [DSL-054](../NORMATIVE.md#DSL-054) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.4, §22.6

---

## Summary

An `InvalidBlockAppeal` whose ground does NOT hold against a genuinely invalid block MUST be Rejected. Covers all four grounds: `BlockActuallyValid` when oracle confirms invalid, `ProposerSignatureInvalid` when signature is valid, `FailureReasonMismatch` when reason matches, `EvidenceEpochMismatch` when epochs match.

---

## Specification

Each ground's predicate is positive; failure to match → `AppealVerdict::Rejected`.

---

## Acceptance Criteria

- [ ] `BlockActuallyValid` + oracle=Invalid → Rejected.
- [ ] `ProposerSignatureInvalid` + valid sig → Rejected.
- [ ] `FailureReasonMismatch` + matching reason → Rejected.
- [ ] `EvidenceEpochMismatch` + matching epochs → Rejected.
