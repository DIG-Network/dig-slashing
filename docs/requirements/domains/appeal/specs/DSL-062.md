# DSL-062: submit_appeal Escrows Appellant Bond

> **Authoritative:** [DSL-062](../NORMATIVE.md#DSL-062) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.1, §2.6, §22.7

---

## Summary

After all structural checks (DSL-055..061, DSL-063), `submit_appeal` MUST lock `APPELLANT_BOND_MOJOS` via `BondEscrow::lock(appellant_index, APPELLANT_BOND_MOJOS, BondTag::Appellant(appeal_hash))`. `BondError::InsufficientBalance` MUST be surfaced as `AppealError::AppellantBondLockFailed`.

---

## Specification

```rust
bond_escrow.lock(
    appeal.appellant_index,
    APPELLANT_BOND_MOJOS,
    BondTag::Appellant(appeal.hash()),
).map_err(|e| AppealError::AppellantBondLockFailed(e.to_string()))?;
```

---

## Acceptance Criteria

- [ ] Successful lock → passes to adjudication.
- [ ] InsufficientBalance → AppellantBondLockFailed.
- [ ] Tag = `BondTag::Appellant(appeal_hash)`.
- [ ] Lock runs AFTER all structural rejections.
