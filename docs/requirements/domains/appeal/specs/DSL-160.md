# DSL-160: SlashAppeal + SlashAppealPayload Serde Roundtrip

> **Authoritative:** [DSL-160](../NORMATIVE.md#DSL-160) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §3.6, §3.7, §18

---

## Summary

`SlashAppeal` + `SlashAppealPayload` enum + all per-offense ground enums (`ProposerAppealGround`, `AttesterAppealGround`, `InvalidBlockAppealGround`) MUST round-trip byte-exactly via bincode + serde_json. Required for REMARK wire (DSL-110) and PendingSlash-history storage.

---

## Specification

Every appeal variant + every ground discriminator must survive encode/decode.

---

## Acceptance Criteria

- [ ] `ProposerSlashingAppeal` + every `ProposerAppealGround` round-trips.
- [ ] `AttesterSlashingAppeal` + every `AttesterAppealGround` (including `ValidatorNotInIntersection { validator_index }`) round-trips.
- [ ] `InvalidBlockAppeal` + every `InvalidBlockAppealGround` round-trips.
- [ ] Witness bytes preserved via `serde_bytes`.
- [ ] Envelope fields (evidence_hash, appellant_index, appellant_puzzle_hash, filed_epoch) preserved.
