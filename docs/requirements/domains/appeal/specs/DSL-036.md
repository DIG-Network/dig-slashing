## Summary

Sustains when the first header's BLS signature does NOT verify under the proposer's pubkey against `dig_block::block_signing_message(network_id, header_a.epoch, &header_a.hash(), header_a.proposer_index)`. Independent re-check using `chia_bls::verify`.

---

## Specification

```rust
let pk = validator_view.get(proposer_index)?.public_key();
let msg = dig_block::block_signing_message(network_id, header_a.epoch, &header_a.hash(), proposer_index);
let sig = Signature::from_bytes(&sig_a_bytes).ok()?;
if !chia_bls::verify(&sig, pk, msg.as_ref()) {
    return AppealVerdict::Sustained { reason: AppealSustainReason::SignatureAInvalid };
}
```

---

## Acceptance Criteria

- [ ] Corrupted sig_a → Sustained.
- [ ] Valid sig_a → Rejected.
- [ ] Sig_a from wrong key → Sustained.
- [ ] Network-id domain binding enforced.
