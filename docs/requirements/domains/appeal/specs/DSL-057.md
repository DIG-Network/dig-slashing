# DSL-057: submit_appeal VariantMismatch

> **Authoritative:** [DSL-057](../NORMATIVE.md#DSL-057) | **Verification:** [VERIFICATION.md](../VERIFICATION.md) | **Tracking:** [TRACKING.yaml](../TRACKING.yaml) | **SPEC:** §6.1, §22.7

---

## Summary

Rejects when `appeal.payload` variant does not match `pending.evidence.payload` variant. E.g., `ProposerSlashingAppeal` against `AttesterSlashing` evidence. Bond NOT locked.

---

## Specification

```rust
match (&appeal.payload, &pending.evidence.payload) {
    (SlashAppealPayload::Proposer(_),     SlashingEvidencePayload::Proposer(_))     => (),
    (SlashAppealPayload::Attester(_),     SlashingEvidencePayload::Attester(_))     => (),
    (SlashAppealPayload::InvalidBlock(_), SlashingEvidencePayload::InvalidBlock(_)) => (),
    _ => return Err(AppealError::VariantMismatch),
}
```

---

## Acceptance Criteria

- [ ] ProposerAppeal vs ProposerSlashing → match.
- [ ] AttesterAppeal vs AttesterSlashing → match.
- [ ] InvalidBlockAppeal vs InvalidBlockProof → match.
- [ ] Any cross-variant combination → VariantMismatch.
- [ ] Bond NOT locked.
