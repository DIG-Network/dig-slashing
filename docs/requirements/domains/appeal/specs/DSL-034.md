## Summary

`ProposerAppealGround::HeadersIdentical` rebuts a `ProposerSlashing` by showing the two `SignedBlockHeader.message` byte-slices are byte-equal. Since `verify_proposer_slashing` (DSL-013) rejects identical headers, a `PendingSlash` with byte-equal headers indicates a buggy verifier and MUST be reverted on appeal.

---

## Specification

```rust
fn verify_proposer_appeal_headers_identical(
    appeal: &ProposerSlashingAppeal,
    evidence: &ProposerSlashing,
) -> AppealVerdict {
    if evidence.signed_header_a.message == evidence.signed_header_b.message {
        AppealVerdict::Sustained { reason: AppealSustainReason::HeadersIdentical }
    } else {
        AppealVerdict::Rejected { reason: AppealRejectReason::GroundDoesNotHold }
    }
}
```

---

## Acceptance Criteria

- [ ] Byte-equal headers → Sustained.
- [ ] Distinct headers → Rejected.
- [ ] Witness field ignored (evidence-only check).
- [ ] Deterministic.
